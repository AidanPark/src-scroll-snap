## 1. 설계 철학 및 원칙

### 1.1 핵심 원칙

| # | 원칙 | 설명 |
|---|------|------|
| 1 | **단일 목적 도구 (Single-Purpose Tool)** | ScrollSnap은 범용 대시보드가 아닌 "캡처 -> 스티칭 -> 다운로드"라는 명확한 한 가지 작업을 수행하는 도구이다. 모든 UI 요소는 이 워크플로우를 가속화하는 데 집중한다. |
| 2 | **상태 머신 기반 UI (State-Driven UI)** | 앱은 명확한 상태 전이를 가지며, 각 상태에서 화면 구성이 달라진다. 사용자는 현재 어떤 단계에 있는지 항상 인지할 수 있어야 한다. |
| 3 | **미리보기 우선 (Preview First)** | 캡처 도구의 핵심은 "지금 무엇이 보이는가"이다. 실시간 비디오 미리보기가 화면의 주인공이며, 컨트롤은 보조 역할이다. |
| 4 | **점진적 공개 (Progressive Disclosure)** | 현재 상태에서 필요하지 않은 컨트롤은 숨긴다. IDLE 상태에서 "종료" 버튼을 보여줄 이유가 없고, CAPTURING 상태에서 "다운로드" 버튼을 보여줄 이유가 없다. |
| 5 | **즉각적 피드백 (Instant Feedback)** | 모든 사용자 액션에 대해 100ms 이내에 시각적 피드백을 제공한다. 장기 작업(조립 등)은 진행률을 실시간으로 표시한다. |
| 6 | **실패에 대한 관용 (Graceful Degradation)** | 에러 상황에서 앱이 멈추지 않고, 사용자에게 복구 경로를 안내한다. |

### 1.2 기존 설계 대비 개선 방향

기존 Phase1 계획서의 UI 설계는 세로 스택(Vertical Stack) 레이아웃이다. 이는 모바일에 적합하지만, 데스크톱 와이드 모니터에서는 다음과 같은 문제가 있다:

| 문제 | 설명 |
|------|------|
| 수평 공간 낭비 | 1920px 폭 모니터에서 중앙에 좁은 컬럼만 사용 |
| 미리보기-컨트롤 분리 | 미리보기와 컨트롤이 세로로 떨어져 동시 확인 불가 |
| 스크롤 강제 | 결과를 보려면 페이지를 아래로 스크롤해야 함 |
| 상태 인지 어려움 | 현재 단계를 파악하려면 여러 UI 요소를 종합해야 함 |

이 제안서는 **2컬럼 레이아웃**을 채택하여 위 문제를 해결한다.

## 2. 전체 레이아웃 아키텍처

### 2.1 2컬럼 레이아웃

OBS Studio, Figma, Adobe Premiere 등 프로 도구의 레이아웃 패턴을 참조한다. 핵심은 **좌측 메인 영역(미리보기/결과) + 우측 사이드바(컨트롤/정보)**의 2컬럼 구조이다.

```
+------------------------------------------------------------------+
| [로고] ScrollSnap  [블록 텍스트] [스크롤 캡처] [파일 수신]              ⚙️ |  <- 탑바 (48px)
+------------------------------------------------------------------+
|                                    |                              |
|                                    |  장치 선택                    |
|                                    |  [v 캡처카드 디바이스 목록]   |
|                                    |                              |
|        메인 콘텐츠 영역             |  스크롤 방향                  |
|        (미리보기 / 결과)            |  [세로] [가로]                |
|                                    |                              |
|        비디오 미리보기              |  ---                         |
|        또는                        |                              |
|        조립 결과 이미지             |  [ ● 캡처 시작 ]             |
|        또는                        |                              |
|        프로그레스 뷰               |  ---                         |
|                                    |                              |
|        (가용 공간 최대 활용)        |  상태 정보                    |
|                                    |  프레임: 0                   |
|                                    |  매칭 품질: --               |
|                                    |                              |
+------------------------------------------------------------------+
| 상태: 대기 중  |  장치: 미연결  |  해상도: --  |  FPS: --       |  <- 상태바 (32px)
+------------------------------------------------------------------+
```

### 2.2 레이아웃 사양

| 영역 | 폭 비율 | 최소 폭 | 설명 |
|------|---------|---------|------|
| 탑바 | 100% | -- | 고정 높이 48px, 앱 제목 + 모드 탭 + 설정 |
| 메인 콘텐츠 | 65~70% | 768px | 미리보기, 결과 이미지, 프로그레스 등 주 컨텐츠 |
| 사이드바 | 30~35% | 320px | 컨트롤, 설정, 상태 정보 |
| 상태바 | 100% | -- | 고정 높이 32px, 연결 상태, 성능 지표 |

### 2.3 반응형 처리

| 뷰포트 | 처리 |
|--------|------|
| 1280px 이상 | 2컬럼 레이아웃 (기본) |
| 1024~1279px | 사이드바 폭 280px로 축소 |
| 1024px 미만 | 지원하지 않음. "데스크톱 브라우저에서 사용해 주세요" 메시지 표시 |

### 2.4 메인 콘텐츠 영역 상태별 변형

메인 콘텐츠 영역은 앱 상태에 따라 다른 콘텐츠를 표시한다:

```
IDLE 상태:
+--------------------------------------+
|                                      |
|     [캡처카드 아이콘]                  |
|                                      |
|     캡처카드를 연결해 주세요            |
|     USB HDMI 캡처카드를               |
|     USB 3.0 포트에 연결하세요          |
|                                      |
|     [장치 감지 중... ]                |
|                                      |
+--------------------------------------+

READY 상태:
+--------------------------------------+
|                                      |
|   +------------------------------+   |
|   |                              |   |
|   |   실시간 비디오 미리보기       |   |
|   |   (16:9 비율 유지)            |   |
|   |                              |   |
|   +------------------------------+   |
|   1920x1080 @ 30fps                  |
|                                      |
+--------------------------------------+

CAPTURING 상태:
+--------------------------------------+
|  ● REC  00:15  [245 frames]          |  <- 캡처 오버레이
|  +------------------------------+   |
|  |  ┌────────────────────┐      |   |
|  |  │ REC 영역 (크롭)    │      |   |
|  |  │ (빨간 테두리 2px)   │      |   |
|  |  │ 외부는 반투명 딤    │      |   |
|  |  └────────────────────┘      |   |
|  +------------------------------+   |
|                                      |
+--------------------------------------+

ASSEMBLING 상태:
+--------------------------------------+
|                                      |

|                                      |

|                                      |
|     처리: 192 / 283 프레임            |
|     체인 길이: 271 프레임              |
|     경과: 0.8초                       |
|                                      |
+--------------------------------------+

RESULT 상태:
+--------------------------------------+
|  [-] [100%] [+]  [화면 맞춤] [원본]   |  <- 뷰어 툴바
|  +------------------------------+   |
|  |                              |   |
|  |   조립된 문서 이미지           |   |
|  |   (스크롤 / 줌 가능)          |   |
|  |                              |   |
|  |                              |   |
|  |                              |   |
|  +------------------------------+   |
|                                      |
+--------------------------------------+
```

## 3. 디자인 시스템

### 3.1 색상 팔레트

다크 테마를 기본으로 채택한다. 이유:
- 캡처 도구 사용 시 눈의 피로를 줄인다
- 비디오 미리보기 영상이 어두운 배경 위에서 더 선명하게 대비된다
- 전문 도구(OBS, DaVinci Resolve 등)의 관행에 부합한다

#### 배경 (Background)

| 용도 | Tailwind 클래스 | Hex | 설명 |
|------|-----------------|-----|------|
| 앱 배경 | `bg-zinc-900` | `#18181b` | 전체 배경 |
| 카드/패널 배경 | `bg-zinc-800` | `#27272a` | 사이드바, 카드 등 |
| 입력/드롭다운 배경 | `bg-zinc-700` | `#3f3f46` | 인터랙티브 요소 |
| 미리보기 영역 배경 | `bg-black` | `#000000` | 비디오가 표시되는 영역 |
| 호버 하이라이트 | `bg-zinc-700/50` | `#3f3f4680` | 마우스 오버 효과 |

#### 텍스트 (Text)

| 용도 | Tailwind 클래스 | Hex | 설명 |
|------|-----------------|-----|------|

| 보조 텍스트 | `text-zinc-400` | `#a1a1aa` | 설명, 부가 정보 |
| 비활성 텍스트 | `text-zinc-500` | `#71717a` | 비활성 상태 |

### 3.2 테두리 및 구분선

| 용도 | Tailwind 클래스 | Hex |
|------|-----------------|-----|
| 기본 구분선 | `border-zinc-700` | `#3f3f46` |

### 3.2.1 스크롤바 스타일

다크 테마와 일관성을 위해 모든 스크롤바에서 흰색 배경을 제거한다.

```css
/* Firefox */
* { scrollbar-color: #3f3f46 transparent; scrollbar-width: thin; }

/* Webkit (Chrome, Edge, Safari) */
::-webkit-scrollbar { width: 6px; height: 6px; }
::-webkit-scrollbar-track { background: transparent; }
::-webkit-scrollbar-thumb { background: #3f3f46; border-radius: 3px; }
::-webkit-scrollbar-thumb:hover { background: #52525b; }
```

| 요소 | 값 | 설명 |
|------|---|------|
| 트랙 배경 | `transparent` | 흰색 배경 제거 |
| 썸 색상 | `#3f3f46` (zinc-700) | 기본 구분선 색상과 통일 |
| 썸 호버 | `#52525b` (zinc-600) | 강조 구분선 색상과 통일 |
| 너비/높이 | 6px | 얇은 스크롤바, `scrollbar-width: thin` |
### 3.3 타이포그래피

시스템 폰트 스택을 사용한다. CDN 의존 없이 모든 OS에서 깔끔하게 렌더링된다.

```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif;

/* 고정폭 (코드, 수치 표시) */
font-family: "SF Mono", "Cascadia Code", "Fira Code", Consolas, monospace;
```

| 용도 | Tailwind 클래스 | 크기 | 굵기 |
|------|-----------------|------|------|
| 앱 제목 | `text-lg font-semibold` | 18px | 600 |
| 섹션 제목 | `text-sm font-semibold uppercase tracking-wide` | 14px | 600 |
| 라벨 | `text-sm font-medium` | 14px | 500 |
| 본문 | `text-sm` | 14px | 400 |
| 수치/코드 | `text-sm font-mono` | 14px | 400 |
| 캡션 | `text-xs` | 12px | 400 |
| 큰 수치 (프레임 수 등) | `text-2xl font-bold font-mono` | 24px | 700 |

### 3.4 간격 토큰

Tailwind CSS의 기본 스페이싱 스케일을 따른다:

| 토큰 | 값 | 용도 |
|------|---|------|
| `p-1` / `gap-1` | 4px | 인라인 요소 간격 |
| `p-2` / `gap-2` | 8px | 아이콘과 라벨 사이 |
| `p-3` / `gap-3` | 12px | 컴팩트 패딩 |
| `p-4` / `gap-4` | 16px | 기본 패딩, 카드 내부 |
| `p-5` / `gap-5` | 20px | 섹션 간 간격 |
| `p-6` / `gap-6` | 24px | 사이드바 섹션 간 간격 |

## 4. Phase 1: 스크롤 캡처 UI/UX

### 4.1 상태 전이도

```
                        장치 감지
              +-------> (자동) -------+
              |                      |
              |                      v
    +------+  |  장치 선택     +--------+     시작 클릭     +-----------+
    | IDLE |--+  완료 -------> | READY  | --------------> | CAPTURING |
    +------+  ^               +--------+ <--+             +-----------+
              |                    ^        |                   |
              |  장치              |  새 캡처 |                   | 종료 클릭
              |  연결 끊김          |  시작   |                   v
              |                    |        |             +------------+
              |                    |        +------------ | ASSEMBLING |
              |                    |                      +------------+
              |                    |                           |
              |                    |         완료              v
              |                    +<-------------------  +--------+
              |                                           | RESULT |
              +<----------------------------------------  +--------+
                             장치 연결 끊김
```

각 상태 전이는 CSS transition (300ms ease)으로 부드럽게 전환한다.

**상태 정의 (진입/이탈/화면):**

| 상태 | 진입 조건 | 이탈 조건 | 핵심 UI |
|------|-----------|-----------|---------|
| IDLE | 앱 초기 진입, 장치 연결 끊김 | 장치 연결 + 스트림 확보 -> READY | 장치 연결 안내, 비활성 컨트롤 |
| READY | 장치 연결 후 미리보기 활성화 | `캡처 시작` 클릭 -> CAPTURING, 장치 연결 끊김 -> IDLE | 라이브 프리뷰, 장치/방향 선택, 캡처 시작 |
| CAPTURING | READY에서 캡처 시작 | 사용자 `캡처 종료` 클릭 -> ASSEMBLING, 장치 연결 끊김 -> 중단 모달 | REC 오버레이, 경과 시간/프레임 수, 매칭 품질 |
| ASSEMBLING | CAPTURING 종료 직후 자동 진입 | 조립 완료 -> RESULT, 사용자 취소 -> READY | 조립 진행률, 캡처/조립 통계, 취소 |
| RESULT | 조립 완료 또는 중단 후 진입 | `새 캡처 시작` 클릭 -> READY, 장치 연결 끊김 -> IDLE | 결과 이미지 뷰어, 다운로드, 새 캡처 |

### 4.2 IDLE 상태 (장치 미연결)

사용자가 앱을 처음 열었을 때 보는 화면이다. 캡처카드가 연결되지 않은 상태.

```
+------------------------------------------------------------------+
| [*] ScrollSnap  [블록 텍스트] [스크롤 캡처] [파일 수신]              ⚙️ |
+------------------------------------------------------------------+
|                                    |                              |
|                                    |  장치                        |
|                                    |  -------------------------   |
|                                    |                              |
|     +----------------------------+ |  연결된 장치 없음              |
|     |                            | |                              |
|     |                            | |  캡처카드를 USB 3.0 포트에    |
|     |      [캡처카드 아이콘]       | |  연결해 주세요.               |
|     |                            | |                              |
|     |  캡처카드를 연결해 주세요     | |  -------------------------   |
|     |                            | |                              |
|     |  USB HDMI 캡처카드를        | |  스크롤 방향                  |
|     |  타겟 PC의 USB 3.0 포트에   | |  [ 세로 | 가로 ]             |
|     |  연결한 후,                 | |                              |
|     |  소스 PC의 HDMI 출력과      | |  -------------------------   |
|     |  캡처카드를 HDMI 케이블로    | |                              |
|     |  연결하세요.                | |  [ ● 캡처 시작 ]  (비활성)    |
|     |                            | |                              |

|     |                            | |                              |

|                                    |                              |
+------------------------------------------------------------------+
| 상태: 장치 대기 중  |  --  |  --  |  --                           |
+------------------------------------------------------------------+
```

**상세 사양:**

| 요소 | 사양 |
|------|------|
| 메인 영역 배경 | `bg-black` 위에 안내 텍스트 중앙 배치 |
| 캡처카드 아이콘 | SVG, 64x64px, `text-zinc-500` |
| 안내 제목 | `text-lg font-medium text-zinc-300` |
| 안내 본문 | `text-sm text-zinc-500 max-w-md text-center leading-relaxed` |
| 감지 중 표시 | 로딩 스피너(12px) + `text-xs text-zinc-500`, 2초 주기로 `enumerateDevices()` 호출 |
| 사이드바 상태 | 장치 드롭다운 비활성, "캡처 시작" 버튼 비활성 (`opacity-50`) |

**자동 전환:** `navigator.mediaDevices.ondevicechange` 이벤트로 장치 연결을 감지하여 자동으로 READY 상태로 전환한다. 전환 시 메인 영역이 300ms fade-in으로 비디오 미리보기로 교체된다.

### 4.4 CAPTURING 상태 (캡처 중)

```

|  +------------------------------+  |  -------------------------   |
|  |                              |  |                              |
|  |                              |  |  경과 시간                    |
|  |                              |  |  00:15                       |
|  |    실시간 비디오 미리보기       |  |                              |
|  |    (빨간 테두리 강조)          |  |  캡처 프레임                  |
|  |                              |  |  245                         |
|  |                              |  |                              |
|  |                              |  |  매칭 품질                    |
|  +------------------------------+  |  [================--] 0.91    |

**상세 사양:**

| 요소 | 사양 |
|------|------|
| REC 표시 | 메인 영역 좌측 상단, `text-red-500 text-sm font-bold`, 원형 아이콘은 1초 주기 깜빡임 (`animate-pulse`) |
| 경과 시간 | REC 표시 우측, `text-sm font-mono text-zinc-300`, MM:SS 형식 |
| 크롭 영역 오버레이 (OCR 수동 설정 시) | 크롭 영역에 `border-2 border-red-500`, 외부는 `boxShadow: '0 0 0 9999px rgba(0,0,0,0.4)'`로 디밍 처리. 좌상단에 "REC 영역" 라벨 표시 (`bg-red-600/80 text-white text-[10px] px-1.5 py-0.5 rounded`) |
| 빨간 테두리 (크롭 미설정 시) | 비디오 요소에 `border-2 border-red-500` 적용, 캡처 시작 시 300ms transition으로 나타남 |
| 사이드바 "캡처 중" | `text-sm font-semibold text-red-400 uppercase tracking-wide`, 좌측에 빨간 원(8px, `animate-pulse`) |

**사이드바 실시간 통계:**

| 항목 | 표시 형식 | 업데이트 주기 |
|------|----------|-------------|
| 경과 시간 | `text-2xl font-bold font-mono` / MM:SS | 매초 |

| 매칭 품질 (NCC confidence) | 프로그레스 바 + 수치 + 텍스트 라벨 | Worker 결과 수신 시마다 |

**매칭 품질 게이지:**

```

```

**연속 실패 경고 (consecutiveFailures >= 5):**

+---------------------------------------------------+

```

**프레임 상한 접근 경고 (frameCount >= 1800):**

|     200프레임 남음. 상한 도달 시 자동 종료됩니다.       |

### 4.7 사이드바 섹션 구조 통합

사이드바는 상태에 따라 섹션이 변한다. 각 섹션은 구분선으로 분리되며, 섹션 제목은 `text-xs font-semibold uppercase tracking-wider text-zinc-500 mb-2`로 통일한다.

| RESULT | 결과 헤더 / 이미지 정보 / (체인 경고) / 다운로드 / 새 캡처 시작 |

## 6. Phase 2: 파일 전송 수신 UI/UX

> 본 섹션은 [컬러 픽셀 그리드 전송 방안](../trans/ScrollSnap_Visual_Transfer_Method.md)을 실안으로 채택한 UI/UX 설계이다.
> 소스 PC에 파일 전달이 불가능하고 키보드 타이핑만 가능한 제약 하에서, **컬러 픽셀 그리드(4색, 96×54)** 방식만을 지원한다.

### 6.1 전체 흐름

```
[소스 PC 안내] ──→ [수신 대기] ──→ [캘리브레이션] ──→ [수신 중] ──→ [완료]
  (SETUP)          (STANDBY)      (자동 진행)       (자동 진행)
```

사용자가 직접 조작하는 단계는 **SETUP**(스크립트 안내 확인 + 수신 시작)과 **COMPLETE**(파일 다운로드)뿐이며, 중간 단계는 자동으로 진행된다.

### 6.2 상태 전이도

```
                        장치 감지
              +-------> (자동) --------+
              |                       |
    +-------+ |                +----------+     픽셀 격자 감지      +-----------+
    | SETUP |--+ 장치 연결 --->| STANDBY  | ────────────────→ | CALIBRATE |
    +-------+ <---+            +----------+ (SC 매직 3연속)    +-----------+
         ^       |                  ^                              |
         |       | 장치 끊김         |  수신 중단                    | 4색 학습 완료
         |       |                  |  클릭                         v
         |       +------------------+                        +-----------+
         |                          +<── 장치 끊김 ──────── | RECEIVING |
         |                                                  +-----------+
         |                                                       |
         |   새 파일 수신                    모든 프레임 수집      |
         +<────────────────────────── +----------+ <────────────+
                                      | COMPLETE |
                                      +----------+
```

각 상태 전이는 CSS transition (300ms ease)으로 부드럽게 전환한다.

**실시간 통계 표시:**

각 수치는 `text-sm font-mono` 스타일이며, 라벨은 `text-xs text-zinc-500`으로 수치 위에 위치한다.

| 항목 | 표시 형식 | 업데이트 주기 |
|------|----------|-------------|

| 남은 시간 | `~MM:SS` | 현재 속도 기반 추정, 10초마다 |

## 6A. Phase 2: 텍스트 캡처 UI/UX

> 본 섹션은 [화면 텍스트 직접 캡처 전송 방안](../trans/ScrollSnap_DirectText_Transfer_Method.md)에 대응하는 UI/UX 설계이다.
> 소스 PC에서 **어떤 사전 작업도 없이** 텍스트 파일을 뷰어로 열고 스크롤하면, 타겟 PC가 캡처 -> 스티칭 -> OCR -> 텍스트 복원을 수행한다.
> 기존 Phase 1(스크롤 캡처) 인프라를 최대한 재활용하고, 텍스트 캡처 특화 UI(영역 선택/속도 피드백/OCR 설정)만 확장한다.

### 6A.1 전체 흐름

```
[소스 PC 안내] ──→ [캡처 중] ──→ [스티칭] ──→ [OCR 처리] ──→ [결과]
  (SETUP)         (CAPTURING)   (STITCHING)  (OCR_PROCESSING) (RESULT)
```

사용자가 직접 조작하는 단계는 **SETUP**(안내 확인 + 캡처 시작)과 **RESULT**(텍스트 편집/저장)이며, 중간 단계는 자동으로 진행된다.

### 6A.2 상태 전이도

```
    +-------+     시작 클릭(장치 연결 시)     +-----------+     종료 클릭     +-----------+
    | SETUP | -----------------------------> | CAPTURING | ---------------> | STITCHING |
    +-------+ <----------------------------- +-----------+                  +-----------+
        ^            새 캡처                        |                            |
        |                                           | 장치 끊김                   | 스티칭 완료
        |                                           v                            v
        |                                      +--------+                +----------------+
        +------------------------------------- | RESULT | <------------- | OCR_PROCESSING |
                       OCR 완료 / 취소          +--------+      취소      +----------------+

SETUP 내부 조건:
- 장치 미연결: 캡처 시작 비활성
- 장치 연결: 캡처 시작 활성
```

각 상태 전이는 CSS transition (300ms ease)으로 부드럽게 전환한다.

### 6A.3 SETUP 상태 (소스 PC 안내 + OCR 타겟 설정)

"텍스트 캡처" 탭을 클릭하면 처음 표시되는 화면. 소스 PC 안내와 함께, OCR 타겟 영역 선택 방식(자동 감지/수동 지정)을 사전 설정한다.

```

|                                    |                              |

|  ② 문서 맨 위(Ctrl+Home) 이동       |  [v AP-HDC4K (#1)         ]  |
|  ③ 캡처 시작 후 천천히 스크롤        |                              |

| 항목 | 사양 |
|------|------|
| 생성 | 드래그 시작점~끝점으로 ROI 생성, 최소 크기 8x4 셀(48×27 기준, 고정밀 모드에서는 16x8 셀) |

`자동 감지` 실패 시 1초 내 인라인 안내(`text-amber-300`)를 표시하고 `수동 지정`으로 포커스를 자동 이동한다.

Cloud Vision 선택 시 제공자/API 키는 **Section 7.1 클라우드 OCR 설정**에서 관리한다. LLM Vision 선택 시 제공자/모델/API 키도 동일 섹션에서 관리한다.

### 6A.4 CAPTURING 상태 (캡처 중 + 스크롤 속도 표시기)

> **Phase 1의 CAPTURING 상태(Section 4.4) 컴포넌트를 재사용**하고, 텍스트 캡처 모드 전용으로 스크롤 속도 표시기를 추가한다.
> 재사용 범위는 프레임 매칭/NCC 품질 지표/캡처 통계의 코어이며, OCR 엔진 배지와 속도 경고 표시는 텍스트 캡처 모드에서 추가한다.

```
메인 오버레이(우상단):
+----------------------------------------+
| 스크롤 속도: 1280 px/s                 |
| [■■■■■■■■■■□□□□]  최적 구간            |
| red=너무 빠름 / green=최적 / yellow=느림 |
+----------------------------------------+
```

**스크롤 속도 표시기 사양:**

| 요소 | Tailwind 클래스 | 동작 |
|------|-----------------|------|
| 속도 카드 | `absolute top-3 right-3 bg-zinc-900/80 border border-zinc-700 rounded-lg px-3 py-2` | 실시간 속도/구간 표시 |
| 구간 바 | `h-2 rounded-full overflow-hidden bg-zinc-700` + 내부 3색 분할 | 빨강/초록/노랑 영역 고정 |
| 현재 포인터 | `w-1.5 h-4 bg-white/90 rounded-full shadow` | 현재 속도 위치 |
| 임계 초과 경고 | `bg-red-900/40 border border-red-700/50 text-red-300 text-xs rounded-md px-2 py-1` | `predictedOverlap < 30%`일 때 노출 |

임계 초과 시 사이드바 상단에 "스크롤이 너무 빠릅니다" 인라인 경고를 즉시 표시한다.

### 6A.5 STITCHING 상태 (스티칭 중)

> **Phase 1의 ASSEMBLING 상태(Section 4.5)와 동일한 컴포넌트를 재사용**한다.

유일한 차이점은 상태바 메시지가 "조립 중"이 아니라 "스티칭 중"으로 표기되는 점이며, 완료 후 RESULT가 아닌 OCR_PROCESSING으로 자동 전환된다.

### 6A.6 OCR_PROCESSING 상태 (OCR 처리 중)

스티칭 완료 후 자동 진입. 긴 스티칭 이미지를 분할해 OCR을 수행한다.

- 제공자별 권장 청크 높이: 1,500~2,000px
- Cloud Vision 권장: 1,500~1,800px (인라인 base64 JSON body 10MB 제한 대응)
- LLM Vision/Tesseract 권장: 1,800~2,000px
- 권장 높이 초과 시: 선택된 제공자 프로파일 기준 분할 + 100px 겹침
- 분할 지점: 줄 사이 빈 영역 우선

예상 OCR 처리 시간(청크당)은 제공자별 범위를 표시한다: Cloud Vision 0.3~0.8초, LLM Vision 0.5~2.6초, Tesseract.js v5 0.8~2.2초.

사이드바에는 OCR 방법, 청크 진행률, 예상 남은 시간을 표시하고 취소 버튼을 제공한다.

### 6A.7 RESULT 상태 (텍스트 결과)

OCR 완료 후 원본 스티칭 이미지와 추출 텍스트를 좌우 분할 뷰로 제공한다.

- 자동 감지/수동 크롭 전환 지원
- 편집 가능 텍스트 패널 제공
- 클립보드 복사 및 확장자별 다운로드 제공

### 6A.8 사이드바 섹션 구조

| 상태 | 사이드바 섹션 |
|------|-------------|
| SETUP | 장치 선택 / 스크롤 방향 / OCR 타겟 영역(자동/수동) / OCR 방법 / OCR 엔진 표시 / 캡처 시작 |
| CAPTURING | 캡처 중 헤더 / OCR 엔진 표시 / 경과 시간 + 프레임 수 / 매칭 품질 / 스크롤 속도 표시기 / (경고) / 캡처 종료 |
| STITCHING | 스티칭 중 헤더 / OCR 엔진 표시 / 캡처 정보 / 스티칭 정보 / 취소 |
| OCR_PROCESSING | OCR 처리 헤더 / OCR 엔진 표시 / OCR 방법 정보 / 이미지 분할 현황 / 진행률 바 / 시간 정보 / 취소 |
| RESULT | OCR 결과 헤더 / OCR 엔진 표시 / 결과 정보 / 크롭 모드 / 다운로드 + 복사 / 새 캡처 시작 |

## 6B. Phase 2: 블록 텍스트 UI/UX

> 본 섹션은 [블록 텍스트 전송 방안](../trans/ScrollSnap_BlockText_Transfer_Method.md)에 대응하는 UI/UX 설계이다.
> 소스 PC 사용자가 텍스트를 마우스로 드래그해 선택하면, 타겟 PC는 롤링 프레임 버퍼 비교와 채도 증가 감지(5단계 파이프라인: 안정성 → 채도증가 → 행트리밍 → 크기 → 밀도)을 통해 텍스트 선택을 자동 감지하고, OCR 수행 후 클립보드로 즉시 전달한다.

### 6B.1 상태 전이도

```
SETUP -> MONITORING <-> PROCESSING -> (back to MONITORING)
                 ^         |
                 +--- TRANSFERRED (toast, then back to MONITORING)
```

**상태 정의 (진입/이탈/화면):**

| 상태 | 진입 조건 | 이탈 조건 | 핵심 UI |
|------|-----------|-----------|---------|
| SETUP | 블록 텍스트 탭 진입, 감지 중단 후 복귀 | `감지 시작` 클릭 + 장치 연결 정상 | 안내 카드, 장치 선택, 임계치 슬라이더, 제공자 요약 |
| MONITORING | SETUP에서 감지 시작, PROCESSING 완료 후 자동 복귀 | 롤링 프레임 비교로 텍스트 선택 감지 시 PROCESSING 진입, `감지 중단` 클릭 시 SETUP 복귀 | 라이브 프리뷰, 감지 상태 인디케이터, 전송 이력, OCR 엔진 배지 |
| PROCESSING | MONITORING에서 텍스트 선택 감지 (안정 + 채도증가 + 밀도통과) | OCR 성공/실패 후 MONITORING 복귀 | ROI 확대 미리보기, OCR 진행 상태, OCR Provider/Model 또는 기능 타입 표시 |
| TRANSFERRED (toast) | PROCESSING 성공 시 즉시 트리거 | 2~5초 토스트 자동 소멸 | 성공 토스트 + 50자 미리보기 + 이력 즉시 추가 |

### 6B.2 SETUP 상태

- OCR 제공자/기능 정보(설정 이동 링크)
- 캡처 간격 선택(0.5s ~ 2.0s, 기본 1.0s)
- `감지 시작` 버튼

```

|                                    |                              |
|  블록 텍스트 안내                   |  감지 설정                    |

| 캡처 간격 슬라이더 | `w-full accent-blue-500` |

| 설정 링크 | `text-sky-400 hover:text-sky-300 underline underline-offset-2` |
| 감지 시작 버튼 | `bg-blue-600 hover:bg-blue-500 text-white h-11 rounded-lg font-medium w-full` |

### 6B.3 MONITORING 상태 (지속 상태, 핵심)

블록 텍스트 모드의 기본 운용 상태. **1초 간격으로 프레임을 캡처하여 롤링 버퍼에 저장하며, 5단계 파이프라인(안정성 게이트 → 채도 증가 감지 → 행 기반 ROI 트리밍 → 영역 크기 → 밀도 검증)을 통해 텍스트 선택을 자동으로 인식한다.** 선택이 감지되면 즉시 OCR을 수행하고 클립보드에 저장한다. 전송 이력을 사이드바에서 관리한다.

MONITORING 상태에서도 사이드바 상단에 현재 OCR 엔진을 고정 표시한다(`OCR: Cloud Vision` / `OCR: GPT-4o` / `OCR: Tesseract.js v5 (eng+kor)`).

```
+------------------------------------------------------------------+
| [*] ScrollSnap  [|||블록 텍스트|||] [스크롤 캡처] [파일 수신]              ⚙️ |
+------------------------------------------------------------------+
|                                    |                              |
|  +------------------------------+  |  감지 상태                   |
|  |   라이브 비디오 프리뷰        |  |  -------------------------   |
||  |                              |  |  상태: 대기 중             |
||  |   ┌──────────────────────┐   |  |  대기 / 변화감지 / 선택감지  |
||  |   │ 감지된 선택 영역      │   |  |  / OCR 처리 중               |
||  |   │ (컴러 보더)          │   |  |                              |
||  |   └──────────────────────┘   |  |  캡처 간격                  |
||  |                              |  |  [ 0.5s --|-- 1.0s --|-- 2.0s ] |
||  |        ( ○ )               |  |                              |
||  |      감지 상태 인디케이터    |  |  전송 이력                   |
||  |  대기 중...                |  |  14:03:21  "const app..."   |
||  |                              |  |  14:01:08  "SELECT * ..."   |
|  +------------------------------+  |  13:57:44  "server { ..."   |
|                                    |                              |
|                                    |  [ 감지 중단 ]               |
+------------------------------------------------------------------+
| 감지 중  |  AP-HDC4K  |  1920x1080  |  대기 중                 |
+------------------------------------------------------------------+
```

**상세 사양:**

| 요소 | Tailwind 클래스 | 설명 |
|------|-----------------|------|
| 감지 보더 | `absolute border-2 border-cyan-400 shadow-[0_0_0_1px_rgba(34,211,238,0.35)]` | 선택 영역 바운딩 시각화 |
| 감지 상태 원형 인디케이터 | `absolute bottom-4 left-4 size-16 rounded-full bg-zinc-900/80 border border-zinc-600 flex items-center justify-center` | 빈 원(대기) → 펴스(변화감지) → 채움(선택확인) |
| 감지 상태 텍스트 | `absolute bottom-4 left-24 bg-zinc-900/80 text-zinc-200 text-xs px-2 py-1 rounded-md` | `대기 중` / `화면 변화 감지` / `선택 영역 감지됨` |
| 상태 배지 | `inline-flex items-center gap-1 px-2 py-0.5 rounded text-xs` + 상태별 색 | 대기/변화감지/선택감지/OCR 처리 |
| 이력 리스트 | `max-h-48 overflow-y-auto divide-y divide-zinc-700` | timestamp + 50자 미리보기 |
| 감지 중단 버튼 | `bg-red-600 hover:bg-red-500 text-white h-10 rounded-lg w-full` | SETUP 복귀 |
| 라이브 화면 토글 | 사이드바 상단 토글 스위치, `라이브 화면` 라벨 | 꺼면: `<video>`는 DOM에 유지하되 `width:0;height:0;position:absolute`로 숨김 (프레임 캡처는 계속 동작). 메인 영역은 컴팩트 상태 패널(감지 상태 + 이력 요약)로 대체 |

**엣지 케이스 처리 (MONITORING):**

| 상황 | UI 동작 |
|------|---------|
| OCR 처리 중 새 선택 발생 | 롤링 버퍼가 계속 동작하므로 새 선택은 자연스럽게 큐잉. 기존 OCR 완료 후 처리 |
| 매우 큰 선택(프레임 35% 이상) | "선택 영역이 큽니다. 처리 시간이 길 수 있습니다" 인라인 경고 + 예상 시간 표시 |
| 선택 해제 감지 | 채도 증가 감지에서 증가 없음으로 판정 → 무시하고 대기 상태 유지 (별도 쿨다운 불필요) |
| 빠른 연속 선택 (2초 이내) | 롤링 버퍼가 자연스럽게 처리 — 불안정 상태에서는 OCR 미실행, 토스트 스팸 방지 1.5초 디바운스 |

전송 이력은 최근 20건만 유지하고, 동일 텍스트(정규화 기준)가 연속 검출될 경우 새 항목을 만들지 않고 기존 항목 timestamp만 갱신한다. 이력 상단에 `이력 지우기` Ghost 버튼을 제공한다.

### 6B.4 PROCESSING 상태 (짧은 자동 상태)

텍스트 선택 감지 시 즉시 진입하며 OCR 완료 시 자동으로 MONITORING으로 복귀한다.

체감 완료 시간은 프레임 버퍼 안정화 + OCR + 클립보드 반영 기준 **약 2.3~4.7초** 범위이며, 일반 사용자 시나리오에서는 **약 2~5초 내** 토스트 확인을 목표로 한다.

```
+------------------------------------------------------------------+
| [*] ScrollSnap  [|||블록 텍스트|||] [스크롤 캡처] [파일 수신]              ⚙️ |
+------------------------------------------------------------------+
|                                    |                              |
|   +----------------------------+   |  OCR 처리 중                 |
|   |   선택 영역 확대 미리보기    |   |  -------------------------   |
|   |   (하이라이트 제거 전처리)   |   |  스피너 + 진행 메시지        |
|   |                            |   |  "선택 영역 OCR 처리 중..." |
|   +----------------------------+   |                              |
|                                    |  완료 후 자동 복귀            |
|                                    |  PROCESSING -> MONITORING    |
+------------------------------------------------------------------+
| OCR 처리 중  |  AP-HDC4K  |  ROI 640x220  |  Cloud Vision / DOCUMENT_TEXT_DETECTION |
+------------------------------------------------------------------+
```

### 6B.5 TRANSFERRED 상태 (토스트)

OCR 성공 시 즉시 표시되는 일시 상태. 상태 자체는 토스트로만 표현되며 화면은 MONITORING을 유지한다.

```
+---------------------------------------------------------------+
| ✅ 클립보드에 복사되었습니다                                  |
| "const extractSelection = async (frame) => { ..."            |
+---------------------------------------------------------------+
```

- 토스트 본문은 앞 50자 미리보기 노출
- 사이드바 전송 이력 즉시 업데이트
- 자동 복귀 없이 MONITORING 지속

### 6B.6 사이드바 섹션 구조

| 상태 | 사이드바 섹션 |
|------|-------------|
| SETUP | 장치 선택 / OCR 제공자 정보 / 캡처 간격 / 감지 시작 |
| MONITORING | OCR 엔진 배지 / **라이브 화면 토글 스위치** / 감지 상태 인디케이터 / 캡처 간격 슬라이더 / 전송 이력 / 감지 중단 |
| PROCESSING | OCR 진행 상태 / 처리 방법(OCR Provider/Model 또는 기능 타입) / OCR 엔진 배지 / 취소(선택) |

## 7. 설정 UI/UX

설정 탭은 전송 모드 공통 환경을 관리한다. 클라우드 OCR API(전용 OCR + LLM Vision), 캡처 기본값, 블록 텍스트 동작을 한 곳에서 제어한다.

### 7.1 클라우드 OCR 설정

```
+------------------------------------------------------------------+
|  클라우드 OCR 설정                                                 |
|------------------------------------------------------------------|
| Provider    | 지원 모델/기능                 | API Key | 연결 테스트 |
| Google Cloud Vision | [DOCUMENT_TEXT_DETECTION] | [••••••] | [ 테스트 ] |
| OpenAI      | [v GPT-4o                 ]   | [••••••] | [ 테스트 ] |
| Anthropic   | [v Claude 3.5 / Claude 4 ]   | [••••••] | [ 테스트 ] |
| Google Gemini | [v Gemini 2.0 / 1.5 Pro ]  | [••••••] | [ 테스트 ] |
|------------------------------------------------------------------|
| 결과: OpenAI 연결 성공 (184ms)                                   |
| [키 삭제] [저장]                                                 |
+------------------------------------------------------------------+
```

| 항목 | 사양 |
|------|------|
| Provider | Google Cloud Vision / OpenAI / Anthropic / Google Gemini |
| 모델/기능 선택 | Google Cloud Vision은 모델 선택 대신 `DOCUMENT_TEXT_DETECTION` 기능 고정 표시, LLM Provider는 모델 드롭다운 (`GPT-4o`, `Claude 3.5 Sonnet`, `Claude 4`, `Gemini`) |
| Cloud Vision 인증 방식 | `API Key(기본)` / `OAuth 2.0 Bearer Token` 세그먼트 제공. OAuth 선택 시 Access Token 입력 필드 노출 |
| API 키 입력 | `type="password"` 마스킹, provider별 별도 보관 |
| Cloud Vision 엔드포인트 | API Key: `POST https://vision.googleapis.com/v1/images:annotate?key=API_KEY` / OAuth: `Authorization: Bearer <access_token>` + `DOCUMENT_TEXT_DETECTION` |
| Cloud Vision 요청 크기 가드 | 인라인 base64 JSON body 10MB 초과 전 사전 차단 + 자동 청크 축소(1,500px 단계) |
| 저장 방식 | `저장 안 함(기본)` / `세션(sessionStorage)` / `이 브라우저에 저장(localStorage)` |
| 연결 테스트 | Google Cloud Vision은 더미 이미지 `TEXT_DETECTION` 호출로 검증, LLM Provider는 소형 샘플 OCR 호출 또는 모델 목록 조회로 검증 |
| 테스트 결과 | 성공/실패 메시지 + 응답 시간 + 오류 원인 표시 |
| 키 삭제 | provider별 `삭제` 버튼 제공 |

**보안/저장 정책:**

- 기본값은 **저장 안 함(세션 메모리만 사용)** 이다.
- 사용자가 명시적으로 선택한 경우에만 `sessionStorage` 또는 `localStorage`에 저장한다.
- `localStorage`는 XSS에 취약하므로, 키 입력 화면에서 스크립트 삽입 지점을 만들지 않는다.
- 공유 PC에서는 사용 후 provider별 키 삭제를 권장한다.
- **Claude Max 구독(claude.ai)과 Anthropic API 사용량은 별도 과금/한도 체계**임을 설정 화면에 명시한다.
- Google Cloud Vision은 API Key 기반 호출을 지원하므로 브라우저 `fetch()` 기반 OCR 경로에 사용 가능하다.
- Google Cloud Vision은 OAuth 2.0 Access Token 기반 호출도 지원하며, OAuth 선택 시 토큰 만료(401) 안내와 재발급 유도 메시지를 노출한다.
- Google Cloud Vision은 Google Gemini(멀티모달 LLM)와 별개 제품이며, Google Cloud Console에서 Vision API 활성화 + API 키 발급이 필요하다.

**연결 테스트/요청 실패 메시지 표준화:**

| 코드/원인 | 사용자 메시지 | 후속 액션 |
|-----------|---------------|-----------|
| 401/403 (키 오류) | "API 키가 유효하지 않습니다." | 키 재입력 버튼 포커스 |
| 429 (Rate Limit) | "요청 한도에 도달했습니다. 잠시 후 다시 시도해 주세요." | 10초 재시도 타이머 표시 |
| Network/Timeout | "네트워크 연결을 확인해 주세요." | `Tesseract` 폴백 버튼 노출 |

## 11. 접근성 고려사항

### 11.1 색상만으로 상태를 표시하지 않기

모든 상태 표시에 색상 외에 추가 시각 정보를 함께 제공한다:

| 정보 | 색상 외 보조 수단 |
|------|------------------|
| 매칭 품질 | 수치(0.91) + 텍스트 라벨("우수") |
| 캡처 중 | 빨간 테두리 + "REC" 텍스트 + 경과 시간 |
| 에러 상태 | 아이콘 + 텍스트 메시지 |
| 프로그레스 | 바 + 퍼센트 수치 |

### 11.2 포커스 관리

| 상황 | 포커스 이동 |
|------|-----------|
| IDLE -> READY 전환 | "캡처 시작" 버튼으로 포커스 |
| CAPTURING -> ASSEMBLING 전환 | 프로그레스 영역으로 포커스 |
| ASSEMBLING -> RESULT 전환 | "PNG 다운로드" 버튼으로 포커스 |
| 모달 열림 | 모달 내부 첫 번째 인터랙티브 요소로 포커스 |
| 모달 닫힘 | 모달을 연 트리거 요소로 포커스 복귀 |

### 11.3 ARIA 라벨

```html
<!-- 비디오 미리보기 -->
<video aria-label="캡처카드 실시간 미리보기">

<!-- 캡처 시작 버튼 -->

     aria-valuenow="0.91" aria-valuetext="우수 (0.91)">

<!-- 프로그레스 바 -->
<div role="progressbar" aria-label="조립 진행률" aria-valuemin="0"
     aria-valuemax="100" aria-valuenow="68">

### 11.4 키보드 탐색

Tab 키 순서는 시각적 레이아웃 순서를 따른다:
1. 탑바 모드 탭 -> 설정 탭
2. 사이드바 컨트롤 (위에서 아래)
3. 메인 영역 인터랙티브 요소 (뷰어 툴바 등)

사이드바 내 섹션은 논리적 그룹으로 묶여 Tab 이동이 자연스럽게 흐른다.

### 11.5 OCR 그리드/블록 텍스트 접근성

| 항목 | 적용 사양 |
|------|-----------|
| OCR ROI 그리드 | `role="application"` + `aria-label="OCR 대상 영역 선택"` + 키보드 이동 안내 툴팁 |
| ROI 핸들 | 각 핸들에 `aria-label` (`좌상단 크기 조절` 등) + `aria-keyshortcuts="ArrowUp ArrowDown ArrowLeft ArrowRight"` |
| OCR 방법 세그먼트(3옵션) | `role="tablist" aria-label="OCR 방법 선택"`, 각 옵션에 `role="tab"` + `aria-selected` 적용 |
| 블록 텍스트 상태 배지 | `role="status" aria-live="polite"`로 상태 변화를 보조기기에 전달 |
| 전송 이력 리스트 | `role="list"`, 항목은 `role="listitem"`, 최신 항목 추가 시 `aria-live="polite"` |
| 토스트 | 성공 토스트는 `aria-live="polite"`, 에러 토스트는 `aria-live="assertive"` 분리 |
