## 1. 개요 및 목적

### 1.1 배경

현재 ScrollSnap은 2컬럼 레이아웃(메인 콘텐츠 65~70% + 사이드바 30~35%)을 사용한다. 캡처카드를 통해 수신하는 원격 화면(소스 PC)은 메인 콘텐츠 영역 내에 `object-fit: contain`으로 표시되어, 실제 비디오가 차지하는 화면 비율이 전체 뷰포트의 약 50~60%에 불과하다.

| 문제 | 설명 |
|------|------|
| 원격 화면 가독성 | 사이드바·탑바·상태바가 화면을 점유하여 원격 화면 표시 영역이 제한적 |
| 텍스트 확인 어려움 | 소스 PC에서 코드나 문서를 읽을 때 글자가 작게 보임 |
| 블록 텍스트 시 선택 영역 확인 | 텍스트 하이라이트 감지 영역을 크게 볼 필요 |

### 1.2 목적

**전체화면 모드**를 제공하여 원격 화면(캡처카드 비디오)이 브라우저 뷰포트를 최대한 채우게 한다. 탑바, 사이드바, 상태바를 숨기고, 꼭 필요한 아이콘만 플로팅으로 제공한다.

### 1.3 핵심 원칙

| # | 원칙 | 설명 |
|---|------|------|
| 1 | **원격 화면 극대화** | 비디오가 뷰포트의 100%를 차지한다. 헤더, 사이드바, 상태바 등 모든 UI 크롬을 숨김. |
| 2 | **최소 플로팅 아이콘** | 전체화면 나가기, 액션(시작/종료), 상태 표시 — 이 3가지만 개별 아이콘으로 플로팅. 바(bar) 형태의 UI를 사용하지 않는다. |
| 3 | **자동 숨김** | 플로팅 아이콘은 마우스 비활동 3초 후 자동 숨김. 마우스 이동 시 즉시 재표시. |
| 4 | **원클릭 토글** | 진입과 해제가 동일한 조작(버튼 또는 단축키 `F`)으로 토글된다. |
| 5 | **상태 보존** | 전체화면 전환 시 캡처 상태, 프레임 데이터, 설정 등 모든 앱 상태가 유지된다. |

## 2. 전체화면 진입 및 해제

### 2.1 진입 방법

| 방법 | 위치 | 조건 |
|------|------|------|
| **전체화면 버튼** | 메인 콘텐츠 영역 우상단 | 비디오가 표시되는 모든 상태 |
| **키보드 단축키 `F`** | 전역 | 비디오가 표시되는 모든 상태, 텍스트 입력 포커스가 아닌 경우 |
| **비디오 더블클릭** | 메인 콘텐츠 영역 비디오 위 | 비디오가 표시되는 모든 상태, **크롭 오버레이가 활성화되지 않은 경우** |

> **⚠️ 더블클릭 충돌 방지**: 스크롤 캡처 모드에서 크롭 영역 설정 오버레이(`CropRectOverlay`)가 활성화된 상태에서는 더블클릭이 크롭 영역 조작으로 해석될 수 있다. `state.scrollCapture.cropRect`이 설정 중이거나 크롭 오버레이 드래그 중인 경우 더블클릭으로의 전체화면 진입을 차단해야 한다. 구현 시 `onDoubleClick` 핸들러에서 크롭 활성 상태를 체크하여 early return 처리한다.

### 2.2 해제 방법

| 방법 | 설명 |
|------|------|
| **Escape 키** | 앱에서 직접 전체화면 해제 |
| **키보드 `F`** | 토글 동작으로 해제 |
| **플로팅 나가기 아이콘** | 우상단 `✕` 아이콘 클릭 |
| **비디오 더블클릭** | 토글 동작으로 해제 |

### 2.3 전체화면 버튼 사양 (일반 모드)

메인 콘텐츠 영역 우상단에 떠있는 반투명 버튼. 비디오 위에 `position: absolute`로 배치한다.

```
위치:    absolute top-3 right-3 z-10
스타일:  bg-zinc-900/70 hover:bg-zinc-900/90 backdrop-blur
         border border-zinc-600/50 rounded-lg
         p-2 text-zinc-300 hover:text-zinc-100
         transition-all duration-150
아이콘:  Maximize (대각 화살표), 20x20px
툴팁:   title="전체화면 (F)"
```

```
+--------------------------------------------+
|                              [⛶]  <- 버튼  |
|   +------------------------------------+   |
|   |                                    |   |
|   |     실시간 비디오 미리보기           |   |
|   |                                    |   |
|   +------------------------------------+   |
+--------------------------------------------+
```

### 2.4 앱 내부 전체화면 메커니즘

브라우저 Fullscreen API를 사용하지 않는다. 앱 내부 상태(`isFullscreen`)만 토글하여 레이아웃을 변경한다.

```javascript
// 진입
dispatch({ type: 'SET_FULLSCREEN', v: true });

// 해제
dispatch({ type: 'SET_FULLSCREEN', v: false });
```

> **참고**: 브라우저 탭바, OS 태스크바는 그대로 유지된다. 비디오는 브라우저 뷰포트(앱 영역) 전체를 채운다. `F11`(브라우저 네이티브 전체화면)과는 별개의 동작이다.

## 3. 전체화면 레이아웃

### 3.1 구조

전체화면 모드에서는 탑바, 사이드바, 상태바를 모두 숨기고, 비디오가 뷰포트 전체를 차지한다. UI는 개별 플로팅 아이콘으로만 제공한다.

```
일반 모드:
+------------------------------------------------------------------+
| [로고] ScrollSnap  [블록 텍스트] [스크롤 캡처] [파일 수신]     ⚙️ |
+------------------------------------------------------------------+
|                                    |                              |
|        메인 콘텐츠 영역             |       사이드바                |
|        (비디오 미리보기)            |       (컨트롤)               |
|                                    |                              |
+------------------------------------------------------------------+
| 상태바                                                           |
+------------------------------------------------------------------+

전체화면 모드:
+------------------------------------------------------------------+
|                                                                   |
|  [상태 뱃지]                                          [✕ 나가기]  |
|                                                                   |
|                                                                   |
|                   캡처카드 비디오                                   |
|                   (뷰포트 100%)                                    |
|                   object-fit: contain                              |
|                   bg-black                                        |
|                                                                   |
|                                                                   |
|                       [액션 버튼]                                  |
|                                                                   |
+------------------------------------------------------------------+
```

> **핵심**: 풀폭 바(bar) 없이, 3개의 독립 플로팅 아이콘만 존재한다.

### 3.2 비디오 배치 사양

| 속성 | 값 | 설명 |
|------|---|------|
| 컨테이너 | `position: fixed; inset: 0; z-index: 50;` | 뷰포트 전체 차지 |
| 배경색 | `bg-black` | 레터박스 영역은 순수 검정 |
| 비디오 | `width: 100%; height: 100%; object-fit: contain;` | 비율 유지, 뷰포트 내 최대 크기 |
| 레터박스 | 자동 (16:9 소스가 16:10 모니터에 표시되면 상하 검정 바) | `object-fit: contain`의 기본 동작 |

### 3.3 Z-index 레이어 구조

| 레이어 | z-index | 내용 |
|--------|---------|------|
| 비디오 | 50 | 캡처카드 비디오 `<video>` |
| 플로팅 아이콘 | 51 | 나가기 버튼, 액션 버튼, 상태 뱃지 |
| 토스트 | 53 | 전체화면 모드에서도 토스트 표시 |
| 모달 | 54 | 장치 끊김 등 긴급 모달 |

> **마이그레이션 필수**: 기존 `ToastContainer`는 `z-50`을 사용 중이다. 전체화면 모드 도입 시 아래 z-index 변경을 반드시 적용해야 한다:
> - `ToastContainer`: `z-50` → `z-[53]`
> - `ModalContainer`: (기존 z-index 없음) → `z-[54]` 추가
> - 기존 코드의 `CropRectOverlay` 등 `z-10`, `z-20`은 전체화면 레이어 아래이므로 변경 불필요.

## 4. 플로팅 아이콘

전체화면 모드에서는 바(bar) 형태의 UI를 사용하지 않는다. 전체화면 나가기, 액션(시작/종료), 상태 표시 — 3가지 기능만 개별 아이콘 버튼으로 화면 위에 플로팅 배치한다.

### 4.1 아이콘 목록

| # | 아이콘 | 위치 | 기능 | 자동 숨김 |
|---|--------|------|------|----------|
| 1 | **나가기 `[✕]`** | 우상단 | 전체화면 해제 | ✓ |
| 2 | **액션 버튼** | 하단 중앙 | 캡처 시작/종료, 감지 중단 | ✓ (CAPTURING 시 항상 표시) |
| 3 | **상태 뱃지** | 좌상단 | REC 표시, 감지 상태 | CAPTURING 시 항상 표시, 그 외 자동 숨김 |

### 4.2 아이콘 배치도

```
+------------------------------------------------------------------+
|                                                                   |
|  [상태 뱃지]                                          [✕ 나가기]  |
|                                                                   |
|                                                                   |
|                   캡처카드 비디오                                   |
|                   (뷰포트 100%)                                    |
|                                                                   |
|                                                                   |
|                       [액션 버튼]                                  |
|                                                                   |
+------------------------------------------------------------------+
```

### 4.3 나가기 버튼 `[✕]`

```
위치:    fixed top-4 right-4 z-[51]
스타일:  bg-zinc-900/60 hover:bg-zinc-900/90 backdrop-blur
         border border-zinc-600/40 rounded-lg
         p-2.5 text-zinc-400 hover:text-white
         transition-all duration-150
아이콘:  XMark, 18px
```

클릭 시 `dispatch({ type: 'SET_FULLSCREEN', v: false })` 호출.

### 4.4 액션 버튼

```
위치:    fixed bottom-8 left-1/2 -translate-x-1/2 z-[51]
스타일:  rounded-full px-5 py-2.5 font-medium text-sm
         shadow-lg shadow-black/40
         transition-all duration-150
```

| 모드 | 상태 | 버튼 내용 | 색상 |
|------|------|----------|------|
| 스크롤 캡처 | READY | `● 캡처 시작` | `bg-blue-600 hover:bg-blue-500 text-white` |
| 스크롤 캡처 | CAPTURING | `■ 캡처 종료  Space` | `bg-red-600 hover:bg-red-500 text-white` — **항상 표시** |
| 블록 텍스트 | MONITORING | `■ 감지 중단` | `bg-zinc-700 hover:bg-zinc-600 text-zinc-200` |
| 블록 텍스트 | PROCESSING | `OCR 처리 중...` | `bg-zinc-700 text-zinc-400 cursor-not-allowed` (비활성) |

> CAPTURING 상태에서는 `Space` 키 힌트를 버튼 내에 `text-xs text-white/60`으로 표시하여 키보드 조작을 안내한다.

### 4.5 상태 뱃지

캡처 진행 중이거나 감지 상태를 표시하는 소형 뱃지. 해당 상태가 없으면 렌더링하지 않는다.

```
위치:    fixed top-4 left-4 z-[51]
스타일:  bg-zinc-900/60 backdrop-blur
         border border-zinc-600/40 rounded-lg
         px-3 py-1.5 text-xs font-mono text-zinc-200
```

| 모드 | 상태 | 뱃지 내용 | 자동 숨김 |
|------|------|----------|----------|
| 스크롤 캡처 | READY | 표시 안 함 | — |
| 스크롤 캡처 | CAPTURING | `● REC  00:15 · 245F` (빨간 dot + 시간 + 프레임수) | **항상 표시** |
| 블록 텍스트 | MONITORING | `○ 대기` / `● 변화감지` / `● 선택감지` (색상 dot) | 자동 숨김 |
| 블록 텍스트 | PROCESSING | `OCR 처리 중...` | 자동 숨김 |

### 4.6 자동 숨김 동작

```
마우스 이동 → 모든 아이콘 표시 (opacity: 1)
                  ↓
              3초 대기 (타이머)
                  ↓
         마우스 이동 없음 → 아이콘 숨김 (opacity: 0, pointer-events: none)
                  ↓
         마우스 이동 감지 → 아이콘 재표시 + 타이머 리셋
```

| 조건 | 동작 |
|------|------|
| 마우스 이동 | 아이콘 즉시 표시, 타이머 3초로 리셋 |
| 3초 비활동 | 300ms fade-out 후 숨김 |
| 아이콘 위에 마우스 호버 | 타이머 일시 정지 (아이콘 유지) |
| 키보드 조작 | 아이콘 표시, 타이머 리셋 |
| CAPTURING 상태 | REC 뱃지 + 종료 버튼 항상 표시 (자동 숨김 제외) |

### 4.7 전체화면에서 접근 불가한 기능

전체화면은 "원격 화면에 집중"하는 모드이다. 아래 기능은 전체화면을 해제한 뒤 일반 모드에서 사용한다.

| 기능 | 접근 방법 |
|------|----------|
| 스크롤 방향 변경 | 전체화면 해제 → 사이드바 |
| OCR 방법 변경 | 전체화면 해제 → 사이드바 |
| 장치 변경 | 전체화면 해제 → 사이드바 |
| API 키 설정 | 전체화면 해제 → 설정 탭 |
| 전송 이력 확인 | 전체화면 해제 → 사이드바 |
| 모드 전환 | 전체화면 해제 → 탑바 |

## 5. 모드별 전체화면 동작

### 5.1 스크롤 캡처 모드

#### READY 상태

```
+------------------------------------------------------------------+
|                                                          [✕]     |
|                                                                   |
|                                                                   |
|                  캡처카드 실시간 비디오                               |
|                  (뷰포트 전체, 16:9 유지)                           |
|                                                                   |
|                                                                   |
|                       [● 캡처 시작]                                |
+------------------------------------------------------------------+
```

- `Space` 키 또는 액션 버튼으로 캡처 시작
- 캡처 시작 시 전체화면 유지, CAPTURING 상태로 전환

#### CAPTURING 상태

```
+------------------------------------------------------------------+
|  ● REC 00:15 · 245F                                     [✕]     |
|                                                                   |
|  +------------------------------------------------------------+  |
|  |                                                            |  |
|  |              캡처카드 실시간 비디오                           |  |
|  |              (빨간 테두리 2px — 크롭 미설정 시)              |  |
|  |                                                            |  |
|  +------------------------------------------------------------+  |
|                                                                   |
|                   [■ 캡처 종료  Space]                            |
+------------------------------------------------------------------+
```

- REC 뱃지(좌상단) + 종료 버튼(하단) — 캡처 중 **항상 표시** (자동 숨김 제외)
- `Space` 키로 캡처 종료 (권장). `Escape`는 전체화면 해제만 수행 (브라우저 제약)
- 캡처 종료 시 → **전체화면 즉시 해제** + ASSEMBLING 상태 전환

#### ASSEMBLING 상태

캡처 종료 시점에 **전체화면을 즉시 해제**하고 ASSEMBLING 상태로 전환한다.

> **근거**: ASSEMBLING 상태는 검은 배경에 프로그레스 바만 표시되어 전체화면의 실익이 없다. 사용자가 "뭔가 잘못된 건가?" 혼란을 느낄 수 있으며, 조립 완료 후 RESULT에서 사이드바 접근이 필요하다. 캡처 종료 시점에 바로 해제하여 전환을 한 번으로 줄인다.

#### RESULT 상태

일반 모드에서 표시. 전체화면은 ASSEMBLING 진입 시 이미 해제됨.

### 5.2 파일 수신 모드

파일 수신 모드에서는 전체화면을 지원하지 않는다. 전체화면 버튼을 숨기고, `F` 키 바인딩도 비활성한다.

> **근거**: 파일 수신 모드의 SETUP 상태는 스크립트 안내 텍스트 중심이며, STANDBY/CALIBRATE/RECEIVING 상태에서는 프레임 맵·캘리브레이션 단계 인디케이터 등 사이드바 정보의 비중이 높아 전체화면의 실익이 적다. 소스 PC 측 F11 전체화면은 기존 안내대로 소스 PC 브라우저에서 수행한다.

### 5.3 블록 텍스트 모드

> **⚠️ 구현 참고: 블록 텍스트 모드의 `<video>` 엘리먼트**
>
> 블록 텍스트 모드는 스크롤 캡처의 `videoRef`와 **별도의 `<video>` 엘리먼트**를 `BlockTextMain` 컴포넌트 내부에서 유지한다 (프레임 캡처용으로 항상 DOM에 존재, `showLiveView` 여부에 따라 시각적으로 숨김 처리).
>
> 전체화면 진입 시 동작:
> - 블록 텍스트의 내부 `<video>`에 동일한 CSS 전용 접근을 적용한다 (`fixed inset-0 z-50 bg-black object-contain`)
> - 블록 텍스트 모드에서는 `BlockTextMain`이 `isFullscreen` prop을 받아 내부 video의 클래스를 전환한다
> - 프레임 캡처용 캐버스 drawing은 동일한 video 엘리먼트를 참조하므로 감지 루프에 영향 없음

#### SETUP 상태

전체화면 진입 불가. 설정 화면이므로 전체화면 의미 없음.

#### MONITORING 상태

```
+------------------------------------------------------------------+
|  ○ 대기                                                  [✕]     |
|                                                                   |
|                  캡처카드 실시간 비디오                               |
|                                                                   |
|              ┌──────────────────────┐                              |
|              │ 감지된 선택 영역      │                              |
|              │ (시안 보더)           │                              |
|              └──────────────────────┘                              |
|                                                                   |
|                      [■ 감지 중단]                                 |
+------------------------------------------------------------------+
```

- 상태 뱃지에 감지 상태를 컬러 dot으로 표시 (`○ 대기` / `● 변화감지` / `● 선택감지`)
- OCR 성공 시 토스트는 전체화면 모드에서도 표시 (z-index: 53)
- 전송 이력은 전체화면에서 접근 불가 — 토스트로 결과를 즉시 확인 가능

> **사용 패턴 참고**: 블록 텍스트는 반복 선택 → OCR → 복사 루프가 잦다. 이력 확인이나 이전 텍스트 재복사가 필요하면 전체화면을 해제해야 한다. 전체화면은 "원격 화면을 크게 보면서 선택 영역을 정확히 잡고 싶을 때" 사용하고, 이력 관리가 중요한 작업에서는 일반 모드가 더 적합하다.

#### PROCESSING 상태

```
+------------------------------------------------------------------+
|  OCR 처리 중...                                          [✕]     |
|                                                                   |
|              +----------------------------+                       |
|              |   선택 영역 확대 미리보기    |                       |
|              |   (하이라이트 제거 전처리)   |                       |
|              +----------------------------+                       |
|                                                                   |
|                    [OCR 처리 중...]                                |
+------------------------------------------------------------------+
```

- PROCESSING 완료 → MONITORING 복귀 (전체화면 유지)
- 토스트로 결과 알림

### 5.4 설정 모드

전체화면 진입 불가. 설정 화면에서는 전체화면 버튼을 숨긴다.

### 5.5 전체화면 가능 여부 요약

| 모드 | 상태 | 전체화면 진입 | 자동 해제 |
|------|------|:---:|:---:|
| 스크롤 캡처 | IDLE | ✕ (비디오 없음) | — |
| 스크롤 캡처 | READY | ✓ | — |
| 스크롤 캡처 | CAPTURING | ✓ | — |
| 스크롤 캡처 | ASSEMBLING | ✕ | ✓ (캡처 종료 시 자동 해제) |
| 스크롤 캡처 | RESULT | ✕ | ✓ (자동 해제) |
| 파일 수신 | 전 상태 | ✕ | — |
| 블록 텍스트 | SETUP | ✕ | — |
| 블록 텍스트 | MONITORING | ✓ | — |
| 블록 텍스트 | PROCESSING | (유지) | — |
| 설정 | — | ✕ | — |

## 6. 상태 전이도

```
                    전체화면 버튼 / F키 / 더블클릭
                         (비디오 표시 상태)
                              │
                              ▼
+-------------+         +------------------+
|  일반 모드   | ------> |  전체화면 모드     |
|  (기본)     | <------ |  (앱 내부 상태)   |
+-------------+    │    +------------------+
                   │         │
                   │         │  ASSEMBLING 진입 시 (캡처 종료)
                   │         │  자동 해제
                   │         ▼
                   │    +------------------+
                   +--- |  일반 모드        |
                        |  (자동 복귀)      |
                        +------------------+

전체화면 해제 트리거:
  1. Escape 키 (앱에서 직접 처리)
  2. F 키 토글
  3. [✕] 아이콘 클릭
  4. 비디오 더블클릭
  5. 캡처 종료 (ASSEMBLING 진입) 시 자동 해제
  6. 장치 연결 끊김 → 자동 해제
```

### 6.1 앱 상태와의 관계

전체화면은 **표시 모드(view mode)**이며, 앱 상태 머신(IDLE → READY → CAPTURING → ...)과 직교한다. 전체화면 전환이 앱 상태를 변경하지 않으며, 앱 상태 전환이 전체화면을 해제하지 않는다 (RESULT 자동 해제 제외).

```javascript
// 앱 상태에 isFullscreen 플래그 추가
var INIT_STATE = {
  // ... 기존 상태 ...
  isFullscreen: false,
};
```

## 7. 트랜지션 및 애니메이션

| 대상 | 속성 | 지속 시간 | 이징 |
|------|------|----------|------|
| 전체화면 진입 | 브라우저 네이티브 (Fullscreen API) | ~200ms | 브라우저 기본 |
| 전체화면 해제 | 브라우저 네이티브 | ~200ms | 브라우저 기본 |
| 플로팅 아이콘 표시 | `opacity` 0→1 | 200ms | ease-out |
| 플로팅 아이콘 숨김 | `opacity` 1→0 | 300ms | ease-in |
| 전체화면 버튼 호버 | `background-color`, `color` | 150ms | ease |

### 7.1 전체화면 전환 시 UI 재배치

```
전체화면 진입:
  1. dispatch({ type: 'SET_FULLSCREEN', v: true })
  2. isFullscreen = true
  3. 탑바, 사이드바, 상태바: display: none (즉시)
  4. 비디오 컨테이너: fixed inset-0 (즉시)
  5. 플로팅 아이콘: opacity 0→1 (200ms fade-in)

전체화면 해제:
  1. dispatch({ type: 'SET_FULLSCREEN', v: false })
  2. isFullscreen = false
  3. isFullscreen = false
  4. 비디오 컨테이너: 원래 위치 복귀 (즉시)
  5. 탑바, 사이드바, 상태바: display 복원 (즉시)
  6. 플로팅 아이콘: 제거
```

## 8. 키보드 단축키

### 8.1 전체화면 모드 전용 단축키

| 단축키 | 동작 | 조건 |
|--------|------|------|
| `F` | 전체화면 토글 | 텍스트 입력에 포커스가 아닌 경우 |
| `Escape` | 전체화면 해제 | 전체화면 상태 |
| `Space` | 캡처 시작/종료 | 스크롤 캡처 모드 READY 또는 CAPTURING |

### 8.2 기존 단축키와의 충돌 방지

| 기존 단축키 | 전체화면 동작 | 충돌 | 해결 |
|------------|-------------|:---:|------|
| `Space` (캡처 토글) | 동일 | ✕ | 동일하게 동작 |
| `Escape` (캡처 종료) | 전체화면 해제 | ✓ | 전체화면 해제를 우선 수행. 캡처 종료는 `Space` 키 권장 — 플로팅 종료 버튼에 `Space` 힌트 표시. 전체화면 해제 직후: ① 사이드바 `[ ■ 캡처 종료 ]` 버튼을 1회 pulse 애니메이션으로 강조, ② **CAPTURING 상태이면 "캡처가 아직 진행 중입니다. Space 키 또는 종료 버튼으로 캡처를 종료하세요." 토스트 알림 자동 표시**. |
| `Ctrl+S` (다운로드) | — | ✕ | RESULT에서는 전체화면 해제됨 |
| `N` (새 캡처) | — | ✕ | RESULT에서는 전체화면 해제됨 |

> **핵심 결정**: 전체화면 모드에서 `Escape`는 항상 **전체화면 해제**를 우선한다. 캡처 종료와 전체화면 해제를 분리하여 혼란을 방지한다: ① CAPTURING 중 플로팅 종료 버튼을 항상 표시하고, ② `Space` 키 힌트를 상시 노출하며, ③ 전체화면 해제 직후 사이드바의 `[ ■ 캡처 종료 ]` 버튼을 1회 pulse 애니메이션으로 강조하고, ④ CAPTURING 상태에서 전체화면이 해제되면 **"캡처 진행 중" 토스트 알림을 자동 표시**하여 사용자 혼란을 방지한다.

#### 8.2.1 전체화면 해제 시 캡처 진행 중 안내 동작

```
isFullscreen 상태 변화 감지 (해제)
  └─ isFullscreen: true → false
       └─ scrollCapture.phase === 'CAPTURING'?
            ├─ YES → ① 토스트 표시: "캡처가 아직 진행 중입니다"
            │        ② 사이드바 종료 버튼 1회 pulse 애니메이션
            │        ③ 토스트 지속시간: 5초 (일반 토스트보다 길게)
            └─ NO  → 별도 안내 없음
```

## 9. 에러 및 엣지 케이스

### 9.1 장치 연결 끊김

| 상황 | 전체화면 동작 |
|------|-------------|
| READY 상태에서 장치 끊김 | 전체화면 자동 해제 + IDLE 전환 |
| CAPTURING 중 장치 끊김 | 전체화면 자동 해제 + 확인 모달 표시 ("조립 시도" / "취소") |
| MONITORING 중 장치 끊김 | 전체화면 자동 해제 + SETUP 복귀 |

### 9.2 Fullscreen API

브라우저 Fullscreen API를 사용하지 않으므로 API 미지원 브라우저에서도 전체화면 기능이 정상 동작한다. 별도 폴백 처리 불필요.

### 9.3 모바일/태블릿

1024px 미만 뷰포트에서는 전체화면 기능을 제공하지 않는다 (기존 반응형 정책과 동일).

### 9.4 다중 모니터

전체화면은 브라우저 창이 위치한 모니터에서 작동한다. 별도 처리 불필요.

### 9.5 전체화면 중 모드 전환

플로팅 아이콘에는 모드 탭을 포함하지 않는다. 모드 전환이 필요하면:
1. 전체화면 해제 (`F` 또는 `Escape`)
2. 탑바에서 모드 전환
3. 전체화면 재진입

이는 의도적 설계이다. 전체화면은 "현재 모드에 집중"하는 용도이며, 모드 전환은 의식적 행위여야 한다.

### 9.6 전체화면 중 설정 변경 필요 시

전체화면에서는 설정 변경 기능을 제공하지 않는다. 설정 변경이 필요하면 전체화면을 해제하고 사이드바 또는 설정 탭에서 수행한다. §4.7 참조.

## 10. 접근성 고려사항

### 10.1 ARIA 라벨

```html
<!-- 전체화면 진입 버튼 (일반 모드) -->
<button aria-label="전체화면 (F)" title="전체화면 (F)">

<!-- 전체화면 나가기 아이콘 -->
<button aria-label="전체화면 나가기 (Escape)" title="전체화면 나가기">

<!-- 액션 버튼 -->
<button aria-label="캡처 시작" title="캡처 시작 (Space)">

<!-- 상태 뱃지 -->
<div role="status" aria-live="polite" aria-label="캡처 상태">
```

### 10.2 포커스 관리

| 상황 | 포커스 이동 |
|------|-----------|
| 전체화면 진입 | 나가기 아이콘으로 포커스 |
| 전체화면 해제 | 전체화면 진입 트리거(버튼)로 포커스 복귀 |

### 10.3 스크린 리더

플로팅 아이콘의 자동 숨김은 시각적 효과(opacity)만 적용하며, DOM에서 제거하지 않는다. `aria-hidden`은 적용하지 않아 스크린 리더에서 항상 접근 가능하다.

```css
/* 자동 숨김 시 */
.overlay-hidden {
  opacity: 0;
  pointer-events: none;
  /* aria-hidden 미적용 — 스크린 리더 접근 유지 */
}
```

## 11. 컴포넌트 트리 변경사항

기존 컴포넌트 트리에 전체화면 관련 컴포넌트를 추가한다.

```
<App>
|
+-- <TopBar />                                   ← 전체화면 시 숨김
|
+-- <MainLayout />
|   +-- <Sidebar />                              ← 전체화면 시 숨김
|   +-- <MainContent />
|       +-- <FullscreenButton />                 ← 추가: 우상단 전체화면 진입 버튼
|       +-- <VideoPreview />                     ← 기존: 전체화면 시 CSS만 변경하여 뷰포트 전체로 확장
|       +-- ... (기존 상태별 뷰)
|
+-- <FullscreenOverlay />                        ← 추가: 전체화면 모드 전용 (isFullscreen일 때만 렌더링)
|   +-- <ExitButton />                              우상단 나가기 아이콘
|   +-- <StatusBadge />                             좌상단 상태 뱃지 (조건부)
|   +-- <ActionButton />                            하단 중앙 액션 버튼
|   (참고: <FullscreenVideo> 별도 컴포넌트 없음 — 기존 <VideoPreview>에 조건부 CSS 적용)
|
+-- <StatusBar />                                ← 전체화면 시 숨김
-- <ToastContainer />                            ← 기존: z-50 → z-[53]으로 변경 필수
-- <ModalContainer />                            ← 기존: z-[54] 추가하여 전체화면에서도 표시
```

> **⚠️ 비디오 재배치 원칙: CSS 전용 (DOM 이동 금지)**
>
> 전체화면 진입 시 기존 `<video>` 요소의 **DOM 위치를 변경하지 않는다**. `<video>` 요소를 `appendChild` 등으로 다른 컨테이너로 이동하면 `srcObject`가 리셋되거나 React 재렌더링 시 예기치 않은 동작이 발생할 수 있다.
>
> 대신, 기존 비디오 컨테이너에 전체화면 전용 CSS 클래스를 **조건부 적용**한다:
>
> ```javascript
> // 비디오 컨테이너 className 결정
> var videoContainerClass = state.isFullscreen
>   ? 'fixed inset-0 z-50 bg-black'  // 전체화면: 뷰포트 전체 차지
>   : 'flex-1 flex flex-col bg-black relative min-h-0';  // 일반: 기존 레이아웃
> ```
>
> 이 접근은 캡처 파이프라인(`captureRef`의 `drawImage`)이 동일한 `videoRef.current`를 참조하므로 캡처 동작에 영향을 주지 않는다.

## 부록 B: 전체화면 가능 조건 체크 함수

```javascript
function canEnterFullscreen(state) {
  // 모드/상태만 체크 (Fullscreen API 미사용)
  // 블록 텍스트 + 스크롤 캡처만 허용
  var mode = state.mode;
  if (mode === MODES.SC) {
    var phase = state.scrollCapture.phase;
    return phase === 'READY' || phase === 'CAPTURING';
  }
  if (mode === MODES.BT) {
    var phase = state.blockText.phase;
    return phase === 'MONITORING' || phase === 'PROCESSING';
  }
  return false;
}

function shouldAutoExitFullscreen(state) {
  var mode = state.mode;
  if (mode === MODES.SC && (state.scrollCapture.phase === 'ASSEMBLING' || state.scrollCapture.phase === 'RESULT')) return true;
  if (!state.device.connected) return true;
  return false;
}
