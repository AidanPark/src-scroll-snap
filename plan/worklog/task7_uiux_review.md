## 검토 내역

### 1차 검토
- 발견된 오류:
  - 일부 와이어프레임 탭바가 5탭 표기(설정 포함)와 불일치(`[@]` 표기)
  - 블록 텍스트/LLM API 신규 에러 상태(미검출, OCR 실패, 클립보드 거부, 키/쿼터/네트워크)가 섹션 10에 누락
  - 블록 텍스트/텍스트 캡처 상태 전이의 진입·이탈 조건 명시가 부족
  - 컴포넌트 트리와 모드 구조 불일치(설정 버튼 중복 개념, OCR ROI 관련 컴포넌트 누락)
  - 상태바 메시지에 블록 텍스트 안정화 단계 메시지 누락
- UX 개선:
  - 텍스트 캡처 수동 ROI 조작(이동/리사이즈/키보드 정밀 이동/초기화) 명세 추가
  - 블록 텍스트 MONITORING 엣지 케이스(새 선택 선점, 대형 선택, 급격 재선택, 미검출 지속) 처리 추가
  - LLM API 키 저장 정책을 기본 미저장으로 조정하고 저장 방식 선택(session/local) 추가
  - 연결 테스트 실패 메시지 표준화(401/403, 429, network/timeout)
  - 접근성 보강(OCR 그리드/핸들 ARIA, 라이브 영역, 토스트 라이브 레벨 분리)
- 수정 내역:
  - 탭바 표기 정정, 상태 정의/에러 처리/접근성/컴포넌트 트리/부록 매트릭스 확장
  - `OCR 탭` 표현을 실제 화면 명칭(텍스트 캡처 결과 화면)으로 정정
  - 관련 문서 목록을 블록 텍스트 문서 기준으로 정정

### 2차 검토
- 발견된 오류:
  - 텍스트 캡처(6A) 상태 전이도는 있었으나 상태별 진입/이탈/핵심 UI 표가 누락
  - 상태바 메시지의 OCR 문구가 모드 맥락 없이 중복되어 해석 여지 존재
  - 키보드 탐색 설명의 `설정 버튼` 용어가 탭 구조와 불일치
- UX 개선:
  - 블록 텍스트 전송 이력 비침습성 강화(최근 20건 유지, 연속 중복 갱신, 이력 지우기)
- 수정 내역:
  - 6A 상태 정의 표 추가
  - 상태바 OCR 문구 명확화(블록 텍스트 OCR 처리 중)
  - 키보드 탐색 문구 정정(설정 탭)
  - 전송 이력 정책/컨트롤 추가

- UX 개선: 없음

- 블록 텍스트 모드 핵심 UX(지속 모니터링, 안정화 인지, 이력 비침습성, 엣지 케이스 대응)를 구체화했다.

- 에러 처리 및 접근성(ARIA/키보드 조작)을 신규 섹션까지 확장해 누락 상태를 메웠다.
