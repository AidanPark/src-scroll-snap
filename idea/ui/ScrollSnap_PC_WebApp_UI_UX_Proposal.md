## 1. 설계 철학 및 원칙

### 1.1 핵심 원칙

| # | 원칙 | 설명 |
|---|------|------|
| 1 | **단일 목적 도구 (Single-Purpose Tool)** | ScrollSnap은 범용 대시보드가 아닌 "캡처 -> 스티칭 -> 다운로드"라는 명확한 한 가지 작업을 수행하는 도구이다. 모든 UI 요소는 이 워크플로우를 가속화하는 데 집중한다. |
| 2 | **상태 머신 기반 UI (State-Driven UI)** | 앱은 명확한 상태 전이를 가지며, 각 상태에서 화면 구성이 달라진다. 사용자는 현재 어떤 단계에 있는지 항상 인지할 수 있어야 한다. |
| 3 | **미리보기 우선 (Preview First)** | 캡처 도구의 핵심은 "지금 무엇이 보이는가"이다. 실시간 비디오 미리보기가 화면의 주인공이며, 컨트롤은 보조 역할이다. |
| 4 | **점진적 공개 (Progressive Disclosure)** | 현재 상태에서 필요하지 않은 컨트롤은 숨긴다. IDLE 상태에서 "종료" 버튼을 보여줄 이유가 없고, CAPTURING 상태에서 "다운로드" 버튼을 보여줄 이유가 없다. |
| 5 | **즉각적 피드백 (Instant Feedback)** | 모든 사용자 액션에 대해 100ms 이내에 시각적 피드백을 제공한다. 장기 작업(조립 등)은 진행률을 실시간으로 표시한다. |
| 6 | **실패에 대한 관용 (Graceful Degradation)** | 에러 상황에서 앱이 멈추지 않고, 사용자에게 복구 경로를 안내한다. |

### 1.2 기존 설계 대비 개선 방향

기존 Phase1 계획서의 UI 설계는 세로 스택(Vertical Stack) 레이아웃이다. 이는 모바일에 적합하지만, 데스크톱 와이드 모니터에서는 다음과 같은 문제가 있다:

| 문제 | 설명 |
|------|------|
| 수평 공간 낭비 | 1920px 폭 모니터에서 중앙에 좁은 컬럼만 사용 |
| 미리보기-컨트롤 분리 | 미리보기와 컨트롤이 세로로 떨어져 동시 확인 불가 |
| 스크롤 강제 | 결과를 보려면 페이지를 아래로 스크롤해야 함 |
| 상태 인지 어려움 | 현재 단계를 파악하려면 여러 UI 요소를 종합해야 함 |

이 제안서는 **2컬럼 레이아웃**을 채택하여 위 문제를 해결한다.

## 2. 전체 레이아웃 아키텍처

### 2.1 2컬럼 레이아웃

OBS Studio, Figma, Adobe Premiere 등 프로 도구의 레이아웃 패턴을 참조한다. 핵심은 **좌측 메인 영역(미리보기/결과) + 우측 사이드바(컨트롤/정보)**의 2컬럼 구조이다.

```
+------------------------------------------------------------------+
| [로고] ScrollSnap  [스크롤 캡처] [파일 수신] [텍스트 캡처] [OCR] [설정] |  <- 탑바 (48px)
+------------------------------------------------------------------+
|                                    |                              |
|                                    |  장치 선택                    |
|                                    |  [v 캡처카드 디바이스 목록]   |
|                                    |                              |
|        메인 콘텐츠 영역             |  스크롤 방향                  |
|        (미리보기 / 결과)            |  [세로] [가로]                |
|                                    |                              |
|        비디오 미리보기              |  ---                         |
|        또는                        |                              |
|        조립 결과 이미지             |  [ ● 캡처 시작 ]             |
|        또는                        |                              |
|        프로그레스 뷰               |  ---                         |
|                                    |                              |
|        (가용 공간 최대 활용)        |  상태 정보                    |
|                                    |  프레임: 0                   |
|                                    |  매칭 품질: --               |
|                                    |                              |
+------------------------------------------------------------------+
| 상태: 대기 중  |  장치: 미연결  |  해상도: --  |  FPS: --       |  <- 상태바 (32px)
+------------------------------------------------------------------+
```

### 2.2 레이아웃 사양

| 영역 | 폭 비율 | 최소 폭 | 설명 |
|------|---------|---------|------|
| 탑바 | 100% | -- | 고정 높이 48px, 앱 제목 + 모드 탭 + 설정 |
| 메인 콘텐츠 | 65~70% | 768px | 미리보기, 결과 이미지, 프로그레스 등 주 컨텐츠 |
| 사이드바 | 30~35% | 320px | 컨트롤, 설정, 상태 정보 |
| 상태바 | 100% | -- | 고정 높이 32px, 연결 상태, 성능 지표 |

### 2.3 반응형 처리

| 뷰포트 | 처리 |
|--------|------|
| 1280px 이상 | 2컬럼 레이아웃 (기본) |
| 1024~1279px | 사이드바 폭 280px로 축소 |
| 1024px 미만 | 지원하지 않음. "데스크톱 브라우저에서 사용해 주세요" 메시지 표시 |

### 2.4 메인 콘텐츠 영역 상태별 변형

메인 콘텐츠 영역은 앱 상태에 따라 다른 콘텐츠를 표시한다:

```
IDLE 상태:
+--------------------------------------+
|                                      |
|     [캡처카드 아이콘]                  |
|                                      |
|     캡처카드를 연결해 주세요            |
|     USB HDMI 캡처카드를               |
|     USB 3.0 포트에 연결하세요          |
|                                      |
|     [장치 감지 중... ]                |
|                                      |
+--------------------------------------+

READY 상태:
+--------------------------------------+
|                                      |
|   +------------------------------+   |
|   |                              |   |
|   |   실시간 비디오 미리보기       |   |
|   |   (16:9 비율 유지)            |   |
|   |                              |   |
|   +------------------------------+   |
|   1920x1080 @ 30fps                  |
|                                      |
+--------------------------------------+

CAPTURING 상태:
+--------------------------------------+
|  ● REC  00:15  [245 frames]          |  <- 캡처 오버레이
|  +------------------------------+   |
|  |                              |   |
|  |   실시간 비디오 미리보기       |   |
|  |   (빨간 테두리 2px)           |   |
|  |                              |   |
|  +------------------------------+   |
|                                      |
+--------------------------------------+

ASSEMBLING 상태:
+--------------------------------------+
|                                      |
|     조립 중...                        |
|                                      |
|     [==============--------] 68%     |
|                                      |
|     처리: 192 / 283 프레임            |
|     체인 길이: 271 프레임              |
|     경과: 0.8초                       |
|                                      |
+--------------------------------------+

RESULT 상태:
+--------------------------------------+
|  [-] [100%] [+]  [화면 맞춤] [원본]   |  <- 뷰어 툴바
|  +------------------------------+   |
|  |                              |   |
|  |   조립된 문서 이미지           |   |
|  |   (스크롤 / 줌 가능)          |   |
|  |                              |   |
|  |                              |   |
|  |                              |   |
|  +------------------------------+   |
|                                      |
+--------------------------------------+
```

## 3. 디자인 시스템

### 3.1 색상 팔레트

다크 테마를 기본으로 채택한다. 이유:
- 캡처 도구 사용 시 눈의 피로를 줄인다
- 비디오 미리보기 영상이 어두운 배경 위에서 더 선명하게 대비된다
- 전문 도구(OBS, DaVinci Resolve 등)의 관행에 부합한다

#### 배경 (Background)

| 용도 | Tailwind 클래스 | Hex | 설명 |
|------|-----------------|-----|------|
| 앱 배경 | `bg-zinc-900` | `#18181b` | 전체 배경 |
| 카드/패널 배경 | `bg-zinc-800` | `#27272a` | 사이드바, 카드 등 |
| 입력/드롭다운 배경 | `bg-zinc-700` | `#3f3f46` | 인터랙티브 요소 |
| 미리보기 영역 배경 | `bg-black` | `#000000` | 비디오가 표시되는 영역 |
| 호버 하이라이트 | `bg-zinc-700/50` | `#3f3f4680` | 마우스 오버 효과 |

#### 텍스트 (Text)

### 3.3 타이포그래피

시스템 폰트 스택을 사용한다. CDN 의존 없이 모든 OS에서 깔끔하게 렌더링된다.

```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif;

/* 고정폭 (코드, 수치 표시) */
font-family: "SF Mono", "Cascadia Code", "Fira Code", Consolas, monospace;
```

| 용도 | Tailwind 클래스 | 크기 | 굵기 |
|------|-----------------|------|------|
| 앱 제목 | `text-lg font-semibold` | 18px | 600 |
| 섹션 제목 | `text-sm font-semibold uppercase tracking-wide` | 14px | 600 |
| 라벨 | `text-sm font-medium` | 14px | 500 |
| 본문 | `text-sm` | 14px | 400 |
| 수치/코드 | `text-sm font-mono` | 14px | 400 |
| 캡션 | `text-xs` | 12px | 400 |
| 큰 수치 (프레임 수 등) | `text-2xl font-bold font-mono` | 24px | 700 |

### 3.4 간격 토큰

Tailwind CSS의 기본 스페이싱 스케일을 따른다:

| 토큰 | 값 | 용도 |
|------|---|------|
| `p-1` / `gap-1` | 4px | 인라인 요소 간격 |
| `p-2` / `gap-2` | 8px | 아이콘과 라벨 사이 |
| `p-3` / `gap-3` | 12px | 컴팩트 패딩 |
| `p-4` / `gap-4` | 16px | 기본 패딩, 카드 내부 |
| `p-5` / `gap-5` | 20px | 섹션 간 간격 |
| `p-6` / `gap-6` | 24px | 사이드바 섹션 간 간격 |

### 3.5 공통 컴포넌트 사양

#### 버튼

| 변형 | 클래스 조합 | 용도 |
|------|------------|------|
| Primary | `bg-blue-600 hover:bg-blue-500 text-white px-4 py-2 rounded-lg font-medium` | 캡처 시작 |
| Danger | `bg-red-600 hover:bg-red-500 text-white px-4 py-2 rounded-lg font-medium` | 캡처 종료 |

```

```

## 4. Phase 1: 스크롤 캡처 UI/UX

### 4.1 상태 전이도

```
                        장치 감지
              +-------> (자동) -------+
              |                      |
              |                      v
    +------+  |  장치 선택     +--------+     시작 클릭     +-----------+
    | IDLE |--+  완료 -------> | READY  | --------------> | CAPTURING |
    +------+  ^               +--------+ <--+             +-----------+
              |                    ^        |                   |
              |  장치              |  새 캡처 |                   | 종료 클릭
              |  연결 끊김          |  시작   |                   v
              |                    |        |             +------------+
              |                    |        +------------ | ASSEMBLING |
              |                    |                      +------------+
              |                    |                           |
              |                    |         완료              v
              |                    +<-------------------  +--------+
              |                                           | RESULT |
              +<----------------------------------------  +--------+
                             장치 연결 끊김
```

각 상태 전이는 CSS transition (300ms ease)으로 부드럽게 전환한다.

### 4.2 IDLE 상태 (장치 미연결)

사용자가 앱을 처음 열었을 때 보는 화면이다. 캡처카드가 연결되지 않은 상태.

```
+------------------------------------------------------------------+
| [*] ScrollSnap  [스크롤 캡처] [파일 수신] [텍스트 캡처] [OCR]  [@] |
+------------------------------------------------------------------+
|                                    |                              |
|                                    |  장치                        |
|                                    |  -------------------------   |
|                                    |                              |
|     +----------------------------+ |  연결된 장치 없음              |
|     |                            | |                              |
|     |                            | |  캡처카드를 USB 3.0 포트에    |
|     |      [캡처카드 아이콘]       | |  연결해 주세요.               |
|     |                            | |                              |
|     |  캡처카드를 연결해 주세요     | |  -------------------------   |
|     |                            | |                              |
|     |  USB HDMI 캡처카드를        | |  스크롤 방향                  |
|     |  타겟 PC의 USB 3.0 포트에   | |  [ 세로 | 가로 ]             |
|     |  연결한 후,                 | |                              |

**상세 사양:**

| 요소 | 사양 |
|------|------|
| 메인 영역 배경 | `bg-black` 위에 안내 텍스트 중앙 배치 |
| 캡처카드 아이콘 | SVG, 64x64px, `text-zinc-500` |

## 6. Phase 2: 파일 전송 수신 UI/UX

> 본 섹션은 [컬러 픽셀 그리드 전송 방안](../trans/ScrollSnap_Visual_Transfer_Method.md)을 실안으로 채택한 UI/UX 설계이다.
> 소스 PC에 파일 전달이 불가능하고 키보드 타이핑만 가능한 제약 하에서, **컬러 픽셀 그리드(4색, 96×54)** 방식만을 지원한다.

### 6.1 전체 흐름

```
[소스 PC 안내] ──→ [수신 대기] ──→ [캘리브레이션] ──→ [수신 중] ──→ [완료]
  (SETUP)          (STANDBY)      (자동 진행)       (자동 진행)
```

사용자가 직접 조작하는 단계는 **SETUP**(스크립트 안내 확인 + 수신 시작)과 **COMPLETE**(파일 다운로드)뿐이며, 중간 단계는 자동으로 진행된다.

### 6.2 상태 전이도

```
                        장치 감지

| 남은 시간 | `~MM:SS` | 현재 속도 기반 추정, 10초마다 |

|     · 소스 PC 화면에 알림/팝업이 없는지 확인           |

|     · DPI 스케일링이 100%인지 확인                    |
+---------------------------------------------------+

       text-sm text-amber-300
```

**특정 프레임 반복 실패 경고 (3루프 연속 실패):**

## 6A. Phase 2: 텍스트 캡처 UI/UX

> 본 섹션은 [화면 텍스트 직접 캡처 전송 방안](../trans/ScrollSnap_DirectText_Transfer_Method.md)에 대응하는 UI/UX 설계이다.
> 소스 PC에서 **어떤 사전 작업도 없이** 텍스트 파일을 뷰어로 열고 스크롤하면, 타겟 PC가 캡처 → 스티칭 → OCR → 텍스트 복원을 수행한다.
> 기존 Phase 1(스크롤 캡처) + Phase 2(OCR) 인프라를 **그대로 재활용**하며, 텍스트 캡처에 특화된 안내 화면과 OCR 파이프라인 UI만 신규 추가한다.

> **탑바 변경**: 본 모드 추가로 탑바 탭이 3개에서 4개로 변경된다.
> 기존: `[스크롤 캡처] [파일 수신] [OCR]`
> 변경: `[스크롤 캡처] [파일 수신] [텍스트 캡처] [OCR]`
> 텍스트 캡처 탭 아이콘은 커서 텍스트(Text Cursor) 아이콘(16px)을 사용한다.

### 6A.1 전체 흐름

```
[소스 PC 안내] ──→ [캡처 중] ──→ [스티칭] ──→ [OCR 처리] ──→ [결과]
  (SETUP)         (CAPTURING)   (STITCHING)  (OCR_PROCESSING) (RESULT)
```

사용자가 직접 조작하는 단계는 **SETUP**(안내 확인 + 캡처 시작)과 **RESULT**(텍스트 편집/저장)뿐이며, 중간 단계는 자동으로 진행된다.

### 6A.2 상태 전이도

```
                        장치 감지
              +-------> (자동) --------+
              |                       |
    +-------+ |                +--------+     시작 클릭     +-----------+
    | SETUP |--+ 장치 연결 --->| SETUP  | ────────────→ | CAPTURING |
    +-------+ <---+           | (장치有) |                +-----------+
         ^       |            +--------+                     |
         |       | 장치 끊김                                  | 종료 클릭
         |       |                                           v
         |       +----+                               +-----------+
         |            |          장치 끊김             | STITCHING |
         |            +<─────────────────────────────  +-----------+
         |                                                  |
         |                                       스티칭 완료  |
         |                                                  v
         |                                          +----------------+
         |   새 캡처         취소                     | OCR_PROCESSING |
         +<─────────── +--------+ <─────────────── +----------------+
                       | RESULT |      OCR 완료          |
                       +--------+ <────────────────────────+
```

각 상태 전이는 CSS transition (300ms ease)으로 부드럽게 전환한다.

### 6A.3 SETUP 상태 (소스 PC 안내)

"텍스트 캡처" 탭을 클릭하면 처음 표시되는 화면. 소스 PC에서 텍스트 파일을 열고 스크롤하는 방법을 안내한다. 파일 수신 모드와 달리 스크립트 타이핑이 전혀 필요 없다.

```
+------------------------------------------------------------------+
| [*] ScrollSnap    [스크롤 캡처] [파일 수신] [|||텍스트 캡처|||] [OCR]  [@] |
+------------------------------------------------------------------+
||                                    |                              |
||     텍스트 캡처 안내                |  소스 PC 설정                 |
||                                    |  -------------------------   |
||  소스 PC의 텍스트를 직접 캡처하여   |                              |
||  타겟 PC에 복원합니다.              |  장치                        |
||  소스 PC에서 어떤 사전 작업도       |  [v AP-HDC4K (#1)         ]  |
||  필요하지 않습니다.                 |                              |
||                                    |  -------------------------   |
||  ① 소스 PC에서 전송할 텍스트 파일을 |                              |
||    뷰어(메모장, VS Code 등)로 열기   |  스크롤 방향                  |
||                                    |  [|||세로|||]  [ 가로 ]       |
||  ② 화면 설정 권장사항               |                              |
||     · 고정폭 폰트 (14pt 이상)       |  -------------------------   |
||     · 밝은 배경 + 어두운 텍스트     |                              |
||     · 줄 번호 끄기                  |  OCR 방법                    |
||     · 줄 바꿈(word wrap) 끄기       |  [|||LLM Vision|||]          |
||     · 미니맵/사이드바 끄기           |  [ Tesseract ]               |
||                                    |                              |
||  ③ 문서 맨 위로 이동 (Ctrl+Home)    |  -------------------------   |
||                                    |                              |
||  ④ 소스 PC 준비 완료 후              |  [ ● 캡처 시작 ]             |
||    아래 "캡처 시작"을 누르고         |                              |
||    소스 PC에서 천천히 스크롤          |  소스 PC 준비 후              |
||                                    |  눌러 주세요.                 |
||  💡 마우스 휠로 초당 3~5줄           |                              |
||    속도로 스크롤하면 최적입니다.      |                              |
||                                    |                              |
+------------------------------------------------------------------+
| 텍스트 캡처 대기  |  AP-HDC4K  |  1920x1080  |  30 fps           |
+------------------------------------------------------------------+
```

**상세 사양:**

| 요소 | 사양 |
|----- -|------|
| 안내 텍스트 | `text-sm text-zinc-300 leading-relaxed` |
| 단계 번호 (①②③④) | `text-base font-bold text-blue-400`, 좌측 정렬 |
| 권장사항 목록 | `text-sm text-zinc-400`, 각 항목 앞에 `·` 마커 |
| 💡 팁 텍스트 | `text-xs text-zinc-500 italic`, 안내 섹션 하단 |
| 사이드바 OCR 방법 | 세그먼트 컨트롤, Section 3.5 세그먼트 컨트롤 스타일과 동일 |
| 캡처 시작 버튼 | `bg-blue-600 hover:bg-blue-500 text-white`, 폭 100%, 높이 44px |
| 캡처카드 미연결 시 | 캡처 시작 버튼 `opacity-50 cursor-not-allowed` |

**OCR 방법 선택 (세그먼트 컨트롤):**

```
컨테이너:   bg-zinc-700 rounded-lg p-0.5 inline-flex gap-0.5 w-full
활성:       bg-blue-600 text-white rounded-md px-3 py-1.5 text-sm font-medium
비활성:     text-zinc-400 rounded-md px-3 py-1.5 text-sm hover:text-zinc-200
```

| 옵션 | 설명 | 조건 |
|------|------|------|
| LLM Vision | 고정밀 OCR (API 키 필요) | API 키 등록 시 기본 선택 |
| Tesseract | 로컬 OCR (무료, 기본 정확도) | API 키 미등록 시 기본 선택 |

API 키 미등록 시 LLM Vision 옵션 선택하면 API 키 설정 모달(Section 7.1)이 자동으로 열린다.

### 6A.4 CAPTURING 상태 (캡처 중)

> **Phase 1의 CAPTURING 상태(Section 4.4)와 완전히 동일한 컴포넌트를 재사용한다.**

캡처 시작 후, 소스 PC에서 문서를 스크롤하는 동안 실시간 비디오 위에 캡처 오버레이가 표시된다. 사이드바에는 경과 시간, 캡처 프레임 수, 매칭 품질이 실시간으로 갱신된다.

유일한 차이점:

| 항목 | 스크롤 캡처 모드 | 텍스트 캡처 모드 |
|------|----------------|----------------|
| 상태바 메시지 | "캡처 중 (N 프레임)" | "텍스트 캡처 중 (N 프레임)" |
| 캡처 종료 후 흐름 | → ASSEMBLING → RESULT (이미지) | → STITCHING → OCR_PROCESSING → RESULT (텍스트) |

와이어프레임, 사이드바 통계, 매칭 품질 게이지, 경고 배너는 모두 Section 4.4와 동일하므로 여기서는 생략한다.

### 6A.5 STITCHING 상태 (스티칭 중)

> **Phase 1의 ASSEMBLING 상태(Section 4.5)와 완전히 동일한 컴포넌트를 재사용한다.**

캡처 종료 후 프레임을 하나의 긴 이미지로 조립하는 상태. 프로그레스 바와 처리 현황이 표시된다.

유일한 차이점:

| 항목 | 스크롤 캡처 모드 | 텍스트 캡처 모드 |
|------|----------------|----------------|
| 상태바 메시지 | "조립 중 (N%)" | "스티칭 중 (N%)" |
| 완료 후 흐름 | → RESULT (이미지 뷰어) | → OCR_PROCESSING (자동 전환) |

스티칭 완료 시 300ms fade 전환으로 OCR_PROCESSING 상태에 자동 진입한다.

### 6A.6 OCR_PROCESSING 상태 (OCR 처리 중)

스티칭 완료 후 자동으로 진입하는 신규 상태. 긴 스티칭 이미지를 청크로 분할하여 OCR을 수행한다. LLM Vision API 또는 Tesseract.js를 사용하며, SETUP 단계에서 선택한 방법으로 처리한다.

```
+------------------------------------------------------------------+
| [*] ScrollSnap    [스크롤 캡처] [파일 수신] [|||텍스트 캡처|||] [OCR]  [@] |
+------------------------------------------------------------------+
||                                    |                              |
||                                    |  OCR 처리 중                  |
||   +------------------------------+ |  -------------------------   |
||   |                              | |                              |
||   |   스티칭 이미지 미리보기       | |  OCR 방법                    |
||   |   (축소, 전체 보기)           | |  LLM Vision (GPT-4o)        |
||   |                              | |                              |
||   |   ┌────────────────────┐     | |  -------------------------   |
||   |   │ ████████████████  │ ◀───┤ |                              |
||   |   │ ████████████████  │ 처리 | |  이미지 분할                  |
||   |   │ ████████████████  │ 중인 | |  총 청크: 5                  |
||   |   │ ████████████████  │ 청크 | |  처리 완료: 2                |
||   |   │ ░░░░░░░░░░░░░░░░  │     | |  처리 중: 3번째              |
||   |   │ ░░░░░░░░░░░░░░░░  │     | |                              |
||   |   │ ░░░░░░░░░░░░░░░░  │     | |  -------------------------   |
||   |   └────────────────────┘     | |                              |
||   |                              | |  진행률                       |
||   +------------------------------+ |  [============------] 60%    |
||                                    |                              |
||   OCR 처리 중... 청크 3/5          |  예상 남은 시간: ~8초          |
||                                    |  경과: 12초                   |
||                                    |                              |
||                                    |  -------------------------   |
||                                    |                              |
||                                    |  [ 취소 ]                    |
||                                    |                              |
+------------------------------------------------------------------+
| OCR 처리 중 (3/5)  |  GPT-4o  |  960x8400  |  60%               |
+------------------------------------------------------------------+
```

**상세 사양:**

| 요소 | 사양 |
|------|------|
| 스티칭 이미지 미리보기 | 메인 영역 내 `object-fit: contain`, `bg-zinc-950 border border-zinc-700 rounded-lg`, 전체 이미지 축소 표시 |
| 처리 완료 영역 | 미리보기 이미지에서 해당 청크 영역을 `bg-emerald-500/20` 반투명 오버레이 |
| 처리 중 영역 | `bg-blue-500/30 animate-pulse` 반투명 오버레이 |
| 미처리 영역 | 원본 이미지 그대로 (오버레이 없음) |
| "OCR 처리 중..." | 미리보기 아래, `text-sm text-zinc-300`, 좌측에 스피너(12px, `animate-spin`) |
| 사이드바 진행률 바 | Section 3.5 프로그레스 바 스타일, `bg-blue-500`, 90%+ 시 `bg-emerald-500` |
| 예상 남은 시간 | `text-sm font-mono text-zinc-400`, 직전 청크 처리 시간 기반 추정 |
| 취소 버튼 | Secondary 스타일, 사이드바 하단 고정. 취소 시 RESULT로 이동하되 부분 OCR 결과만 표시 |

**이미지 분할 기준:**

```
스티칭 이미지 높이 ≤ 2,000px → 분할 없이 단일 OCR
스티칭 이미지 높이 > 2,000px → 2,000px 단위 분할
  각 청크는 하단 100px 겹침 포함 (줄 경계에서 분할)
  분할 지점: 밝기 변화가 적은 수평 라인 (줄 사이 빈 공간)
```

**OCR 에러 처리:**

| 에러 | 처리 |
|------|------|
| API 호출 실패 (네트워크) | 토스트 경고 + 3회 자동 재시도. 3회 실패 시 Tesseract 폴백 제안 |
| API 키 만료/무효 | 토스트 에러 + API 키 설정 모달 자동 열기 |
| Tesseract 로딩 실패 | 토스트 에러: "OCR 엔진을 로드할 수 없습니다" |
| 청크 OCR 부분 실패 | 성공한 청크만으로 부분 결과 표시, 실패 청크 위치 안내 |

### 6A.7 RESULT 상태 (텍스트 결과)

OCR 완료 후 좌우 분할 뷰로 원본 스티칭 이미지와 추출된 텍스트를 동시에 표시한다. Section 7.2(OCR 실행 화면)의 분할 뷰와 유사하지만, 텍스트 영역 크롭과 후처리 기능이 추가된다.

```
+------------------------------------------------------------------+
| [*] ScrollSnap    [스크롤 캡처] [파일 수신] [|||텍스트 캡처|||] [OCR]  [@] |
+------------------------------------------------------------------+
||                          |                 |                      |
|| 원본 이미지               | <-> 드래그     | 추출된 텍스트          |
|| +-----------------------+|  가능한        | +-------------------+ |
|| |                       ||  구분선        | | def hello():      | |
|| |  스티칭된 텍스트        ||               | |     print("Hi")   | |
|| |  이미지                ||               | |                   | |
|| |                       ||               | | def main():       | |
|| |  (현재 보고 있는 영역   ||               | |     hello()       | |
|| |   파란 하이라이트)      ||               | |                   | |
|| |                       ||               | | if __name__ == .. | |
|| |                       ||               | |     main()        | |
|| +-----------------------+|               | +-------------------+ |
||                          |               |                      |
|| 텍스트 영역:              |               | [ 보기 | 편집 ]      |
|| [자동 감지] [수동 크롭]    |               |                      |
||                          |               | [복사] [다운로드 ▾]   |
+------------------------------------------------------------------+
| OCR 완료 (12.3초)  |  GPT-4o  |  2,847 문자  |  142줄             |
+------------------------------------------------------------------+
```

**분할 뷰 사양:**

| 요소 | 사양 |
|------|------|
| 좌측 패널 (원본 이미지) | 기본 50%, 최소 30%, Section 7.2와 동일 |
| 우측 패널 (추출 텍스트) | 기본 50%, 최소 30% |
| 구분선 | `w-1 bg-zinc-700 cursor-col-resize hover:bg-blue-500`, 드래그로 비율 조정 |
| 동기 스크롤 | 좌측 이미지 스크롤 시 우측 텍스트 해당 위치 자동 스크롤 |
| 이미지-텍스트 연동 | 좌측 이미지 영역 클릭 → 우측 텍스트 하이라이트, 반대도 동일 |

**텍스트 영역 크롭 (좌측 패널 하단):**

에디터 UI 요소(제목 바, 상태 바, 줄 번호 등)를 제외하고 텍스트 본문만 추출하기 위한 기능.

| 모드 | 동작 |
|------|------|
| 자동 감지 (기본) | 스티칭 이미지에서 텍스트 본문 영역을 자동 감지. 상단/하단 UI 바, 좌측 줄 번호 영역, 우측 스크롤바를 자동 제외 |
| 수동 크롭 | 이미지 위에 드래그로 직사각형 영역 지정. 선택 영역만 OCR 대상 |

```
자동 감지 배지:  bg-emerald-900/50 text-emerald-400 px-2 py-0.5 rounded text-xs
수동 크롭 배지:  bg-blue-900/50 text-blue-400 px-2 py-0.5 rounded text-xs

수동 크롭 드래그 영역:
  선택 중:  border-2 border-blue-400 border-dashed bg-blue-400/10
  확정 후:  border-2 border-blue-500 bg-blue-500/5
  비선택:  bg-black/50 (어둡게 딤 처리)
```

크롭 영역 변경 후 "OCR 재실행" 버튼이 활성화된다.

**텍스트 편집 패널 (우측):**

| 요소 | 사양 |
|------|------|
| 편집 영역 | `contenteditable="true"` 또는 `<textarea>`, `font-mono text-sm leading-relaxed bg-zinc-800 p-4 text-zinc-100` |
| 보기/편집 토글 | 세그먼트 컨트롤, 보기 모드에서는 구문 강조 렌더링 |
| 문자 수/줄 수 | 에디터 하단 우측, `text-xs font-mono text-zinc-500` |

**다운로드 드롭다운:**

```
+-------------------------+
| 텍스트 (.txt)            |
| Python (.py)            |
| JavaScript (.js)        |
| JSON (.json)            |
| YAML (.yaml)            |
| Markdown (.md)          |
| 그대로 저장 (확장자 지정) |
+-------------------------+
```

확장자 선택 시 해당 확장자로 파일을 다운로드한다. 파일명 자동 생성: `text_capture_YYYYMMDD_HHmmss.{ext}`

**OCR 재실행:**

텍스트 영역 크롭 변경, OCR 방법 변경, 또는 부분 실패 시 OCR을 재실행할 수 있다.

```
재실행 버튼: bg-zinc-700 hover:bg-zinc-600 text-zinc-100 px-3 py-1.5 rounded-lg text-sm
위치: 좌측 패널 하단 (크롭 모드 옆) 또는 사이드바
```

### 6A.8 사이드바 섹션 구조

사이드바는 상태에 따라 섹션이 변한다. 각 섹션은 구분선으로 분리되며, 섹션 제목은 `text-xs font-semibold uppercase tracking-wider text-zinc-500 mb-2`로 통일한다.

| 상태 | 사이드바 섹션 |
|------|-------------|
| SETUP | 장치 선택 / 스크롤 방향 / OCR 방법 / 캡처 시작 버튼 |
| CAPTURING | 캡처 중 헤더 / 경과 시간 + 프레임 수 / 매칭 품질 / (경고) / 캡처 종료 |
| STITCHING | 스티칭 중 헤더 / 캡처 정보 / 스티칭 정보 / 취소 |
| OCR_PROCESSING | OCR 처리 헤더 / OCR 방법 정보 / 이미지 분할 현황 / 진행률 바 / 시간 정보 / 취소 |
| RESULT | OCR 결과 헤더 / 결과 정보 (문자 수, 줄 수, 모델, 시간) / 크롭 모드 / 다운로드 + 복사 / 새 캡처 시작 |

**이미지-텍스트 연동:**

원본 이미지에서 특정 영역을 클릭하면, 해당 영역에서 인식된 텍스트가 우측 패널에서 하이라이트된다. 반대로 텍스트를 클릭하면 해당 텍스트가 추출된 이미지 영역이 좌측에서 하이라이트된다.

## 9. 인터랙션 디자인 상세

### 9.1 키보드 단축키

| `Ctrl + -` | 결과 뷰어 줌 아웃 | RESULT 상태 |
| `Ctrl + 0` | 결과 뷰어 화면 맞춤 | RESULT 상태 |
| `N` | 새 캡처 시작 | RESULT 상태 |

### 9.2 버튼 상태

모든 버튼은 다음 4가지 시각 상태를 가진다:

| 상태 | 처리 |
|------|------|
| 기본 (Default) | 정의된 기본 스타일 |
| 호버 (Hover) | 밝기 +10%, `transition-colors duration-150` |
| 액티브 (Active/Pressed) | 밝기 -5%, `transform scale-[0.98]`, `transition duration-75` |
| 비활성 (Disabled) | `opacity-50 cursor-not-allowed`, 클릭 이벤트 무시 |

### 9.3 트랜지션 사양

| 대상 | 속성 | 지속 시간 | 이징 |
|------|------|----------|------|
| 버튼 색상 변경 | `background-color`, `color` | 150ms | ease |
| 상태별 메인 영역 전환 | `opacity` | 300ms | ease-in-out |
| 사이드바 섹션 전환 | `opacity`, `max-height` | 250ms | ease |
| 프로그레스 바 채움 | `width` | 300ms | ease-out |
| 매칭 품질 바 색상 | `background-color` | 300ms | ease |
| 토스트 진입 | `transform`, `opacity` | 300ms | ease-out |
| 토스트 퇴장 | `transform`, `opacity` | 200ms | ease-in |
| 캡처 중 REC 깜빡임 | `opacity` (1 -> 0.3 -> 1) | 1000ms | ease-in-out, infinite |
| 뷰어 줌 | `transform: scale()` | 200ms | ease-out |

## 10. 에러 및 엣지 케이스 UI

### 10.1 캡처카드 미인식

| 상황 | UI 처리 |
|------|---------|
| USB 연결 없음 | IDLE 상태 유지, 메인 영역에 연결 안내 표시 |
| 장치 감지되나 스트림 실패 | 토스트 에러: "캡처카드에 접근할 수 없습니다. 다른 프로그램이 사용 중이거나 브라우저 권한을 확인해 주세요." |
| 권한 거부 | 메인 영역에 권한 안내: "카메라 접근 권한이 필요합니다. 브라우저 주소창의 카메라 아이콘을 클릭하여 허용해 주세요." |

### 10.2 캡처 중 장치 연결 끊김

```
+---------------------------------------------+
|  캡처카드 연결이 끊어졌습니다                   |
|                                             |
|  캡처가 자동으로 중단되었습니다.                |
|  지금까지 캡처된 245개 프레임으로 조립을         |

|  [ 조립 시도 ]         [ 취소 (데이터 폐기) ] |

이 경우 확인 모달로 처리한다. "조립 시도"를 선택하면 현재까지의 데이터로 ASSEMBLING 상태로 전환.

### 10.3 매칭 연속 실패

인라인 경고 배너(사이드바)로 처리. 섹션 4.4에서 정의한 스타일 사용.

연속 실패가 10회에 도달하면 경고 배너를 강화:

```

|                                                   |
|     [캡처 종료하고 조립]  [계속 캡처]                 |
+---------------------------------------------------+
스타일: bg-red-900/30 border border-red-700/50 rounded-lg p-3
```

+---------------------------------------------+

### 10.5 브라우저 호환성 경고

앱 로드 시 브라우저 호환성을 검사한다.

| 검사 항목 | 미지원 시 |
|----------|----------|
| `getUserMedia` API | 앱 사용 불가 안내 |
| `requestVideoFrameCallback` | 토스트 경고: "이 브라우저에서는 프레임 동기화가 정확하지 않을 수 있습니다. Chrome 또는 Edge를 권장합니다." + 자동 `requestAnimationFrame` 폴백 |
| `OffscreenCanvas` (Web Worker용) | 폴백 사용, 별도 안내 없음 |
| Chromium 기반 여부 | 상태바에 "Chrome 또는 Edge 권장" 표시 |

### 10.6 메모리 부족

조립 중 Canvas 생성 실패 시:

```
토스트 에러: "메모리가 부족하여 조립에 실패했습니다. 해상도를 50%로 낮추어 재시도합니다."
자동으로 해상도를 50%로 낮추어 재시도. 재시도도 실패 시:
토스트 에러: "메모리 부족으로 조립할 수 없습니다. 브라우저의 다른 탭을 닫고 다시 시도해 주세요."
```

## 11. 접근성 고려사항

### 11.1 색상만으로 상태를 표시하지 않기

모든 상태 표시에 색상 외에 추가 시각 정보를 함께 제공한다:

| 정보 | 색상 외 보조 수단 |
|------|------------------|
| 매칭 품질 | 수치(0.91) + 텍스트 라벨("우수") |
| 캡처 중 | 빨간 테두리 + "REC" 텍스트 + 경과 시간 |
| 에러 상태 | 아이콘 + 텍스트 메시지 |
| 프로그레스 | 바 + 퍼센트 수치 |

### 11.2 포커스 관리

| 상황 | 포커스 이동 |
|------|-----------|
| IDLE -> READY 전환 | "캡처 시작" 버튼으로 포커스 |
| CAPTURING -> ASSEMBLING 전환 | 프로그레스 영역으로 포커스 |
| ASSEMBLING -> RESULT 전환 | "PNG 다운로드" 버튼으로 포커스 |
| 모달 열림 | 모달 내부 첫 번째 인터랙티브 요소로 포커스 |
| 모달 닫힘 | 모달을 연 트리거 요소로 포커스 복귀 |

### 11.3 ARIA 라벨

```html
<!-- 비디오 미리보기 -->
<video aria-label="캡처카드 실시간 미리보기">

<!-- 캡처 시작 버튼 -->
<button aria-label="스크롤 캡처 시작">

<!-- 매칭 품질 게이지 -->
<div role="meter" aria-label="매칭 품질" aria-valuemin="0" aria-valuemax="1"
     aria-valuenow="0.91" aria-valuetext="우수 (0.91)">

<!-- 프로그레스 바 -->
<div role="progressbar" aria-label="조립 진행률" aria-valuemin="0"
     aria-valuemax="100" aria-valuenow="68">

<!-- 상태바 -->
<footer role="status" aria-live="polite">

<!-- 토스트 알림 -->
<div role="alert" aria-live="assertive">
```

### 11.4 키보드 탐색

Tab 키 순서는 시각적 레이아웃 순서를 따른다:
1. 탑바 모드 탭 -> 설정 버튼
2. 사이드바 컨트롤 (위에서 아래)
3. 메인 영역 인터랙티브 요소 (뷰어 툴바 등)

사이드바 내 섹션은 논리적 그룹으로 묶여 Tab 이동이 자연스럽게 흐른다.
